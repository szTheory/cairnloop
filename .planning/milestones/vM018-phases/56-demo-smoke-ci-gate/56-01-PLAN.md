<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub event to workflow | Untrusted push and PR metadata selects whether Docker smoke runs. |
| Pull request code to runner | PR code executes `./bin/demo smoke` on a GitHub-hosted runner. |
| Workflow to release state | CI configuration can accidentally gain write or publishing capabilities. |
| Workflow to Docker diagnostics | Docker build and smoke logs may expose environment or secrets if secrets are passed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-56-01 | Elevation of privilege | `.github/workflows/demo-smoke.yml` pull_request event | mitigate | Use `pull_request`, `permissions: contents: read`, no `pull_request_target`, no `contents: write`, and no `secrets.*`; enforce with `Cairnloop.DemoSmokeWorkflowContractTest`. |
| T-56-02 | Tampering | Release publishing boundary | mitigate | Keep demo smoke in `.github/workflows/demo-smoke.yml` only; do not modify `release-please.yml`; contract test rejects release and publish tokens in the smoke workflow. |
| T-56-03 | Information disclosure | Docker smoke workflow environment and logs | mitigate | Do not pass OpenAI, Hex, release, or external service secrets; rely on `./bin/demo smoke` inline route/log diagnostics without adding secret-bearing env. |
| T-56-04 | Denial of service | Docker build and health wait on CI runner | mitigate | Keep `timeout-minutes: 25`, Docker and Compose version preflight, and wrapper-owned bounded waits and cleanup. |
| T-56-05 | Repudiation | Branch-protection-ready check identity | mitigate | Keep stable job id/name `demo-smoke` so CI evidence has a durable check name. |
| T-56-06 | Tampering | Path filter drift | mitigate | Source contract test asserts the exact D-05 push and pull_request path lists and rejects `.planning/**`. |
| T-56-07 | Spoofing | Workflow command drift | mitigate | Source contract test asserts exact `run: ./bin/demo smoke` and rejects raw Compose or route-level curl smoke duplication in YAML. |
| T-56-SC | Tampering | package installs | accept | No npm, pip, cargo, Mix, or action dependency additions are planned; existing `actions/checkout@v4` remains the only action dependency in this workflow. |
</threat_model>
