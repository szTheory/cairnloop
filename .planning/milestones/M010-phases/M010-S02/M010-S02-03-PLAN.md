| Boundary | Description |
|----------|-------------|
| suggestion domain -> operator review UI | Durable AI-prepared content becomes operator-facing workflow state |
| evidence/citation snapshot -> review copy | UI language must preserve canonical-vs-assistive trust semantics |
| review surface -> KB editor | Manual editing must remain an explicit operator handoff, not an implicit publish shortcut |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-M010-S02-07 | S | `SuggestionReview` action set | mitigate | Expose only inspect, regenerate, dismiss, and open-for-manual-edit actions in Phase 10, and omit publish or approval controls entirely from the route and template. |
| T-M010-S02-08 | T | suggestion diff / trust presentation | mitigate | Derive revision diffs from `base_revision_id` plus `proposed_markdown`, and route all trust/citation wording through a presenter so assistive evidence cannot masquerade as canonical grounding. |
| T-M010-S02-09 | E | editor handoff | mitigate | Require an explicit `open_for_manual_edit` action from the review surface before the existing editor loads suggestion content, preserving the review boundary and preventing generation from acting like direct authoring. |
</threat_model>
