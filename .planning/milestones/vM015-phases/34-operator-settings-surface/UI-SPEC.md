## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--cl-bg)` | App/page background |
| Secondary (30%) | `var(--cl-surface)` | Cards, panels, input fields |
| Accent (10%) | `var(--cl-primary)` | Primary CTAs, active route/tab, focus rings |
| Destructive | `var(--cl-danger)` | Revoke actions, error text, blocked policy |
| Success/Health | `var(--cl-success)` | Healthy Notifiers, synced pgvector index |

Accent reserved for: Primary action buttons ("Add MCP Token", "Save Policy"), active state on tabs/navigation, and focus rings. Never use it for full-page backgrounds or large decorative elements.

## Interaction & State Specifics (Phase 34)

- **Dark Mode (SET-04):** Must persist via a mechanism that Phoenix respects (e.g. cookie, localStorage) or Ecto state, applying `data-theme="dark"` to the document/body so that CSS variables from `cairnloop.css` instantly re-evaluate.
- **MCP Tokens (SET-01):** Display tokens in a list. Tokens must be visually masked (`cl_mcp_***`) by default. Include a "Revoke" button with destructive confirmation.
- **Health Indicators (SET-02, SET-03):** Use small visual dot/badge motifs next to Notifier and Retrieval subsystems. Green (`--cl-success`) for healthy/connected. Red (`--cl-danger`) for unreachable/desynced. Amber (`--cl-warning`) for pending.
- **Settings Layout:** Evolve `SettingsLive` to an operator cockpit pattern (left rail navigation or stacked `.cl-card` panels) rather than flat headers, organizing SLA Policies, System Health, and Authentication tokens distinctively.
- **Form UI:** Use standard inputs with `var(--cl-border)` borders and `var(--cl-focus)` focus rings.
