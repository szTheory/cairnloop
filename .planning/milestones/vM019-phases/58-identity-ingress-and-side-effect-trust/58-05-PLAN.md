<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Cairnloop config -> side-effect bridge | host opt-in controls whether external automation can run |
| conversation telemetry -> Oban job | generic telemetry event may attempt to enqueue external side effect |
| Oban worker -> Scrypath API | durable support data leaves the host app only after explicit opt-in and ready config |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-58-05-01 | Information Disclosure | Application Scrypath bridge | mitigate | no enqueue unless ScrypathConfig.status/1 is ready |
| T-58-05-02 | Tampering | IngestScrypath worker | mitigate | reject missing/unsafe default URL/key before Req client construction |
| T-58-05-03 | Denial of Service | misconfigured side-effect config | mitigate | deterministic no-enqueue/error state and doctor-facing bounded reasons in Plan 58-07 |
| T-58-05-SC | Tampering | package installs | accept | no new npm/pip/cargo package installs in this plan |
</threat_model>
