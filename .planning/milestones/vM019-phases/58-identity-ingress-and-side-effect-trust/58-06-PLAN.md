## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| domain workflow -> telemetry exporters | support workflow facts leave durable DB path for observability |
| email ingress/worker -> Logger | inbound support payloads could appear in application logs |
| resolved conversation event -> optional side-effect bridge | side-effect bridge receives only a durable pointer, not raw support content |
