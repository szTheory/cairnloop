<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host router session -> ConversationLive | host-owned operator identity enters dashboard process |
| ConversationLive -> Chat/Outbound/Governance | operator action crosses into durable workflow/audit state |
| ConversationLive render -> operator | fail-closed trust state is shown without sensitive detail |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-58-03-01 | Spoofing | ConversationLive mutation handlers | mitigate | session actor guard blocks mutation without host_user_id |
| T-58-03-02 | Repudiation | Governance approvals and resolve/recovery events | mitigate | actor comes from dashboard session only, never customer_ref or conversation.host_user_id |
| T-58-03-03 | Information Disclosure | trust-state UI | mitigate | humanized copy and disclosure discipline from 58-UI-SPEC |
| T-58-03-SC | Tampering | package installs | accept | no new npm/pip/cargo package installs in this plan |
</threat_model>
