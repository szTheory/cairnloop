<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser widget -> WidgetSocket | untrusted customer/session token enters Cairnloop |
| WidgetSocket -> WidgetChannel | only verified customer_ref may cross into channel state |
| WidgetChannel -> Chat/Conversation | browser identity becomes durable customer_ref, not operator actor |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-58-02-01 | Spoofing | WidgetSocket.connect/3 | mitigate | configured verifier required before assigning customer_ref |
| T-58-02-02 | Elevation of Privilege | WidgetChannel.join/3 and Chat.create_customer_conversation/1 | mitigate | customer token writes customer_ref only; host_user_id remains operator/governance identity per D-01 |
| T-58-02-03 | Repudiation | example demo config | mitigate | demo/test verifier is explicit config, not implicit token acceptance |
| T-58-02-SC | Tampering | package installs | accept | no new npm/pip/cargo package installs in this plan |
</threat_model>
