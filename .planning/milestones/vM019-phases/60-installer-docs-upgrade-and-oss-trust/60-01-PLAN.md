<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Maintainer docs edit -> public adopter | Untrusted or stale prose can become adopter setup instructions. |
| Public docs -> host app setup | Incorrect commands/config can alter host DB schemas, auth posture, or operational expectations. |
| Package metadata -> HexDocs/Hex package | Missing docs/assets or stale version snippets undermine package trust. |
| Security policy -> vulnerability reporter | Public-reporting guidance can leak exploit details if wrong. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-60-01 | Tampering | install docs source scans | mitigate | Reject stale prefix and migration guidance before docs changes per D-04/D-05/D-13. |
| T-60-02 | Repudiation | package/docs source scans | mitigate | Assert package files, ExDoc extras, version snippets, and asset references from `mix.exs` and disk per D-11/D-13. |
| T-60-03 | Information Disclosure | SECURITY.md reporting guidance | mitigate | Require private reporting and reject public exploit-disclosure instructions per D-08. |
| T-60-04 | Denial of Service | docs quality gates | mitigate | Keep tests DB-free and focused so guardrails run quickly before `mix ci.fast` and `mix ci.quality`. |
| T-60-SC | Tampering | package installs | accept | No new npm, pip, cargo, Hex, or Mix dependency install task is planned; Package Legitimacy Audit says no new package. |
</threat_model>
