<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Public install docs -> host DB commands | Adopters may copy commands directly into production-like host apps. |
| Public upgrade docs -> existing public-schema install | Incorrect data-move or rollback guidance can damage support-domain data. |
| Troubleshooting docs -> production operators | Overstated doctor/health claims can hide real readiness failures. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-60-05 | Tampering | README/Quickstart/Troubleshooting migration commands | mitigate | Source-scan public docs for installer-aligned ordered migrations and reject stale broad-prefix instructions per D-05. |
| T-60-06 | Spoofing | Host auth/operator identity docs | mitigate | Keep host route auth and operator identity injection explicit per D-06 and Phase 58 summaries. |
| T-60-07 | Denial of Service | Upgrade data-move guidance | mitigate | Require maintenance-window, backup, verification, and rollback guidance in UPGRADING per D-07. |
| T-60-08 | Information Disclosure | Troubleshooting/doctor docs | mitigate | Keep `/health` liveness-only and direct richer checks to doctor without claiming unchecked DB/Oban/pgvector/Scrypath readiness. |
| T-60-SC | Tampering | package installs | accept | No package install task is planned; use existing Mix/ExUnit/ExDoc/Hex tooling. |
</threat_model>
