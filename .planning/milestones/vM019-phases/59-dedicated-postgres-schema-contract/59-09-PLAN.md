| Boundary | Description |

| raw retrieval SQL -> Postgres | Retrieval health checks use SQL strings and must qualify only Cairnloop-owned tables. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-59-40 | Tampering | raw retrieval table identifiers | mitigate | Use `Cairnloop.SchemaPrefix.quoted_table/2` for Cairnloop tables. |
| T-59-41 | Tampering | query prefix precedence | mitigate | Apply explicit from/join prefixes where `@schema_prefix` or Repo opts would be insufficient. |
| T-59-42 | Information Disclosure / Integrity | silent public/dedicated split | mitigate | Add collision tests with same-name public rows. |
| T-59-43 | Denial of Service / Integrity | Oban queue checks | mitigate | Keep `oban_jobs` and `Oban.Job` host-owned and test that they are not schema-prefixed by Cairnloop config. |
| T-59-44 | Denial of Service | bulk chunk refreshes | mitigate | Assert prefix behavior on `Ecto.Multi.insert_all/delete_all` so chunks are not deleted/inserted in the wrong schema. |
</threat_model>
