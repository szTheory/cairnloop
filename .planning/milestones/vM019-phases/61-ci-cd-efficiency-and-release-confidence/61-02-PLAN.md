## Trust Boundaries

| Boundary | Description |
| --- | --- |
| PR source -> CI jobs | Untrusted changes can influence source but must not receive write or release credentials. |
| CI artifacts -> maintainer diagnostics | Failure evidence must be useful without leaking secrets, support content, or long logs. |

| --- | --- | --- | --- | --- |
| T-61-04 | Tampering | GitHub Actions action pins | mitigate | Use maintained first-party majors verified from official sources and lock them with source-contract tests. |
| T-61-05 | Denial of Service | E2E runner time | mitigate | Add source-controlled path gating and aggregate gate semantics for intentional skips. |
| T-61-06 | Information Disclosure | Playwright artifacts and summaries | mitigate | Capture failure-only traces/screenshots with short retention and bounded summaries. |
| T-61-SC | Tampering | npm/pip/cargo installs | accept | No new package-manager dependency is added; existing `npm ci` remains in the example E2E lane. |
