## Trust Boundaries

| Boundary | Description |
| --- | --- |
| Trusted main/manual release -> publish job | Only trusted refs may reach Hex publish credentials. |
| Release workflow source -> irreversible publish | Source must prove exact SHA and package/docs readiness before publish. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --- | --- | --- | --- | --- |
| T-61-10 | Elevation of Privilege | `release-please` job | mitigate | Keep write permissions scoped only to release PR automation and keep checkout credentials unpersisted. |
| T-61-11 | Tampering | `publish-hex` release SHA | mitigate | Check out `needs.release-please.outputs.sha` and run `mix ci.quality` before publish. |
| T-61-12 | Information Disclosure | `HEX_API_KEY` and release summary | mitigate | Preflight secret presence without printing value; summaries include bounded metadata only. |
| T-61-SC | Tampering | npm/pip/cargo installs | accept | This plan adds no new package-manager dependency. |
</threat_model>
