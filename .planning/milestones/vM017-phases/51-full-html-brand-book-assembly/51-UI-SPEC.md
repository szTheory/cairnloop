# Phase 51 — UI Design Contract

> Visual and interaction contract for the full standalone HTML brand book. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | static HTML plus Cairnloop `--cl-*` tokens; no framework runtime |
| Icon library | inline local symbols only for status meaning; no icon package |
| Font | Atkinson Hyperlegible for UI/body, Fraunces for display specimens, Martian Mono for token names and file paths |

Sources: `.planning/STATE.md`, `.planning/ROADMAP.md`, `.planning/REQUIREMENTS.md`, Phase 47/48/50 UI specs, Phase 49 logo decisions, Phase 50 scaffold decisions, `brandbook/assets/css/tokens.css`, `brandbook/color/swatches.json`, `logo/USAGE.md`, `prompts/cairnloop_brand_book.md`, and `.planning/phases/48-token-evolution-lock-propagate/48-CONTRAST-REVERIFY.md`.

No `components.json`, React, Next.js, Vite, shadcn preset, Tailwind config, or local TSX component inventory is present. Do not initialize shadcn for this phase. The brand book remains a repo-local, self-contained static document that opens directly from `file://`.

## Phase UI Boundary

Phase 51 upgrades the Phase 50 scaffold into the complete HTML brand book. It must consume existing token, contrast, logo, voice, imagery, and motion sources; it must not reopen brand selection or wire assets into shipped surfaces.

| Surface | Contract |
|---------|----------|
| `brandbook/index.html` | Complete single-page brand reference with all required sections rendered as live HTML |
| `brandbook/assets/css/brandbook.css` | Full brandbook layout, tables, specimens, diagrams, theme toggle states, and responsive behavior using `--cl-*` tokens |
| `brandbook/assets/css/tokens.css` | Derived token source; read from it and verify with the Phase 50 derivation script; do not hand-edit |
| `brandbook/color/swatches.json` | Source data for color swatches; keep it lean and generated, not a contrast authority |
| `brandbook/logo/` and `brandbook/raster/` | Relative brandbook copies or links to committed logo/raster assets; no remote downloads |
| `logo/USAGE.md` | Source for approved files, clearspace, minimum sizes, and do/don't rules |
| Phase 48 contrast evidence | Source for WCAG-AA badge text and pass/exempt rows |

Out of scope: editing canonical token values, changing the selected C3.6 logo family, adding new logo assets, wiring README/example app/favicon/OG metadata, changing `mix.exs` package files, adding a Phoenix route, or adding a build pipeline.

## Content Architecture

Use one ordered page with a sticky in-page navigation on desktop and a compact anchored contents list on mobile. Do not create a marketing landing page; the first viewport must identify `Cairnloop brand book`, show provenance, expose the light/dark toggle, and hint at the first token section.

| Section | Required live HTML treatment |
|---------|------------------------------|
| Header | Compact title, tagline `Support that leaves a trail.`, provenance line, `Network dependency: none`, and theme toggle |
| Contents | Anchor links to Color, Typography, Spacing/Radius/Shadow/Motion, Logo, Voice/Microcopy, Imagery, Motion, Downloads |
| Color | Swatch groups for primitive, semantic light, and semantic dark tokens; every swatch shows visible color, token name, hex value, role, theme, and WCAG status badge |
| Typography | Real-font specimens for `--cl-font-title`, panel, body, small, micro, and code; each row shows token name, pixel size, line height, family, and intended use |
| Spacing/radius/shadow/motion | Token tables reading from `tokens.css`; visual rulers for spacing, radius chips, shadow elevation samples, and motion timing rows |
| Logo system | Lockup gallery at multiple sizes; approved file table; clearspace diagram; minimum-size table; no-cage and no-icon-left-of-text rules; relative download links to committed SVG/raster assets |
| Voice and microcopy | Rendered voice pillars, do/avoid examples, product naming rules, CTA examples, empty/error/status copy examples, not prose-only placeholders |
| Imagery | Approved/forbidden image categories as visual panels using local CSS shapes, route-line motifs, and text labels; no stock or remote imagery |
| Motion | Motion principles, allowed timing/easing tokens, reduced-motion note, and static specimens showing route/state/progress use without autoplay dependency |
| Footer | Regeneration commands, package boundary, source files, and Phase 52 wiring boundary |

No section may be a stub. Each section must include at least one concrete rendered example, one source/provenance cue, and one usage rule.

## Layout Contract

| Area | Required treatment |
|------|--------------------|
| Page shell | Full-width `--cl-bg` canvas; constrained content max-width `1200px`; page gutter `16px` mobile and `32px` desktop |
| Header | Dense reference-document header, not hero-scale marketing composition; use the SVG logo as an asset sample, not a text-recreated wordmark |
| Navigation | Desktop nav may be sticky; mobile nav is static/anchor-based and must not cover content |
| Sections | Full-width unframed sections with constrained inner content; individual repeated examples may use cards with radius <= `var(--cl-radius-sm)` |
| Tables | Responsive tables must keep token names, values, and badges readable; allow horizontal scroll only inside the table wrapper |
| Galleries | Logo proof rows use fixed rendered sizes: 16px, 24px, 48px, 112px minimum lockup, and 256px specimen where relevant |
| Diagrams | Clearspace/min-size diagrams are CSS/SVG/HTML diagrams using token-driven surfaces and labels, not screenshot images |

Do not use nested cards, decorative gradient/orb backgrounds, dark blurred imagery, oversized hero type, or stock-like media. The document should feel like a precise OSS maintainer reference.

## Spacing Scale

Declared values use the approved Cairnloop 4px rhythm. Layout CSS for Phase 51 must use these values unless it is displaying a derived token value in a specimen.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline token/value gaps, badge gaps, icon/text spacing |
| sm | 8px | Compact rows, swatch label gaps, table cell internals |
| md | 16px | Default block spacing, mobile page gutter, card padding |
| lg | 24px | Section padding, specimen group spacing, logo proof row gaps |
| xl | 32px | Desktop grid gaps and section rhythm |
| 2xl | 48px | Major vertical breaks between top-level sections |
| 3xl | 64px | Top/bottom page padding and large reference groups |

Exceptions: fixed logo proof sizes `16px`, `24px`, `48px`, `112px`, and `256px`; minimum focusable target `44px`; copied token specimens may display canonical values such as `2px`, `12px`, `20px`, and `40px` because they are derived token content, not new layout rhythm.

## Typography

Use exactly these four UI sizes and two weights for the brandbook interface. Token specimens may display canonical token values as content.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Metadata / token code | 12px | 400 | 1.5 |
| Body | 15px | 400 | 1.6 |
| Section heading | 18px | 600 | 1.44 |
| Page title / display proof | 28px | 600 | 1.29 |

Typeface usage:

| Typeface | Usage |
|----------|-------|
| Atkinson Hyperlegible Next / Atkinson Hyperlegible / system sans fallback | Body, labels, navigation, badges, voice examples, tables |
| Fraunces / Georgia fallback | Page title, display specimens, tagline specimens only |
| Martian Mono / Atkinson Hyperlegible Mono / system mono fallback | Token names, hex values, file paths, code snippets, ratio badges |

No network font loading, font binaries, CDN calls, negative letter spacing, viewport-scaled type, or live text recreation of the logo wordmark is allowed. The committed SVG wordmark is the only wordmark rendering source.

## Color

Use the evolved Refined palette from `tokens.css`; do not create a parallel brandbook palette.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F4EEE2` light canvas, `#101614` dark canvas | Page background and broad brandbook canvas |
| Secondary (30%) | `#FAF5EB` light surface, `#FFFFFF` light raised surface, `#141B19` dark surface, `#1F2C28` dark raised surface | Tables, specimen panels, swatch cells, logo proof surfaces |
| Accent (10%) | `#A8492A` light copper, `#D98A4A` dark copper | Theme toggle active state, focus ring, current nav marker, C3.6 ring reference, selected/current badges, primary download link emphasis |
| Destructive | `#B54C36` light danger, `#C96A55` dark danger | Error states, forbidden do/don't panels, failed verification only |

Accent reserved for: focus rings, current nav marker, active theme toggle segment, C3.6 ring reference, selected/current badges, route-marker annotations, and one primary download/link emphasis per section. Do not use copper for every heading, every link, broad backgrounds, decorative fills, table borders, or non-selected panel chrome.

All meaningful text pairings must meet WCAG AA 4.5:1. Meaningful focus indicators, swatch boundaries, and selected/current indicators must meet 3.0:1. Status/error/selected states must always pair color with visible text or an accessible icon label. Contrast badges must read as `AA pass`, `UI pass`, or `Decorative exempt`; never rely on green/red alone.

## Token Rendering Contract

| Token group | Required rendering |
|-------------|--------------------|
| Color primitives | Swatch, token name, raw value, display hex, role, and theme label from `swatches.json` |
| Light/dark semantics | Separate light and dark groupings; each row shows semantic role and resolved display hex |
| Contrast badges | Render Phase 48 pass/exempt outcomes for core text, UI boundary, focus, status chip, and copper route-marker pairings |
| Typography | Specimen text using the actual font stack and declared token size/line-height values from `tokens.css` |
| Spacing | Visual ruler row for each `--cl-space-*` token; label both token and pixel value |
| Radius | Corner specimen for each `--cl-radius-*` token; show token and value |
| Shadow | Elevation sample for `--cl-shadow-1` through `--cl-shadow-4` and semantic aliases |
| Motion | Table for duration/easing tokens; static route/progress examples must include reduced-motion guidance |

If JavaScript reads CSS variables at runtime, it must read only local computed styles and local JSON. If JavaScript is unavailable, the core document content must remain visible and the theme defaults to light.

## Interaction Contract

| Interaction | Contract |
|-------------|----------|
| Local load | `index.html` opens from `file://` with no console errors and no failed or remote network requests |
| Theme toggle | Two-state `Light` / `Dark` control, keyboard accessible, at least 44px high, using `aria-pressed` or equivalent state text; works with local JavaScript only |
| Theme persistence | Optional `localStorage` persistence is allowed; if blocked/unavailable, the toggle still changes the current document session |
| Navigation | Anchor links are fragment-only; current/active state must have text or structural cue in addition to color |
| Downloads | Relative links to local committed assets; no remote fetch, blob generation, analytics, or beacon |
| Tables | Horizontal overflow is contained within table wrappers; focus outlines remain visible while scrolling |
| Motion | No autoplay animation required. Hover/focus transitions are <= 180ms except static motion-token specimens; use transform/opacity/color only and honor `prefers-reduced-motion` |
| Failure handling | Missing JSON/assets produce visible inline copy naming the failed local asset and next action; page must not go blank |

No external scripts, iframes, analytics, CDN assets, remote fonts, package registries, or browser network dependencies are permitted.

| Element | Copy |
|---------|------|
| Primary CTA | Download logo assets |

## Voice, Microcopy, Imagery, And Motion Guidance

| Area | Required live rendering |
|------|-------------------------|
| Voice pillars | Render clear examples for precise, calm, protective, and OSS-native copy; include "use" and "avoid" pairs |
| Naming | Show `Cairnloop`, `:cairn_loop`, and `CairnLoop` in distinct prose/package/module examples |
| Microcopy | Include CTA, empty, error, warning, success, and handoff examples that pair status color with text/icon labels |
| Imagery | Approved categories: trail markers/cairns, topographic routes, material textures, operator workspace, product UI composites, abstract support loops |
| Forbidden imagery | Explicitly label no headset agents, call centers, chat bubbles, robots, purple neural gradients, wellness pebble stacks, fantasy maps, and generic SaaS people |
| Motion | Explain route/state/progress use, transform/opacity-only rule, reduced-motion support, and forbidden bouncy/chatbot/glowing motion |

Guidance sections must use structured examples, comparison panels, checklists, or specimens. They must not be prose-only paragraphs.

## Verification Expectations

The planner/executor should verify this UI contract with source, derivation, and browser checks.

| Check | Required evidence |
|-------|-------------------|
| Required sections | `brandbook/index.html` contains Color, Typography, Spacing, Radius, Shadow, Motion, Logo, Voice, Microcopy, Imagery, and Downloads sections |
| Token derivation | `mix run scripts/derive_brandbook_tokens.exs --check` passes before and after assembly |
| Live token rendering | Color swatches, type specimens, spacing/radius/shadow/motion tables read from `tokens.css` and/or generated token data, not a hand-maintained palette |
| Contrast badges | Core AA/UI/decorative outcomes from Phase 48 evidence render with text labels and do not rely on color alone |
| Logo gallery | Every approved Phase 49 logo/raster asset has a rendered specimen and relative download link |
| Usage diagrams | Clearspace, minimum-size, do, and don't panels are visible and include no-cage and no-icon-left-of-text rules |
| Local browser | Browser opens `file://.../brandbook/index.html` with no console errors, page errors, failed requests, or remote network requests |
| Theme toggle | Automated browser proof toggles light/dark, verifies document state changes, and confirms the toggle remains keyboard/focus visible |
| Relative paths | Source grep proves no `http://`, `https://`, protocol-relative `//`, CDN, `@import`, external `url(...)`, iframe, beacon, or root-relative production path dependency |
| Accessibility | Status/selected/error indicators include text or icon labels; all focusable controls meet 44px target and visible focus requirements |
| Package hygiene | `mix.exs` package `files` list does not include `brandbook/` |
| Scope guard | Diff is confined to `brandbook/`, verification helpers/tests, and Phase 51 planning artifacts unless a plan explicitly authorizes otherwise |

Rendered-behavior checks that need a browser must be automated with the existing Playwright file-url approach; do not create human UAT tasks for console/network/theme-toggle proof.
