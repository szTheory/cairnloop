<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo sources -> generated HTML | Local Markdown/JSON/CSS/planning evidence is transformed into committed HTML; malformed or missing inputs must fail closed. |
| brandbook HTML -> local filesystem assets | `file://brandbook/index.html` resolves stylesheets and `../logo/*` downloads without remote requests. |
| brandbook collateral -> Hex package | Repo-local collateral must remain unshipped and outside `mix.exs` package files. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-51-01 | Tampering | `scripts/assemble_brandbook.exs` | mitigate | Deterministic `--check` compares generated `brandbook/index.html` bytes and fails loudly on drift, missing inputs, or missing source sections. |
| T-51-02 | Spoofing | Logo gallery/download inventory | mitigate | Source tests assert every download link points to an approved committed `../logo/*` asset from `logo/USAGE.md`; no redraw/recompose task is allowed. |
| T-51-03 | Information Disclosure | Static brandbook dependencies | mitigate | Forbidden dependency scan blocks HTTP(S), protocol-relative paths, imports, iframes, beacons, and root-relative asset paths. |
| T-51-04 | Repudiation | Contrast/status claims | mitigate | Tests require visible text labels `AA pass`, `UI pass`, and `Decorative exempt` from Phase 48 evidence so claims are reviewable in committed HTML. |
| T-51-SC | Tampering | npm/pip/cargo installs | accept | No new package installs are planned; existing locked Playwright and Jason dependencies are reused. |
</threat_model>
