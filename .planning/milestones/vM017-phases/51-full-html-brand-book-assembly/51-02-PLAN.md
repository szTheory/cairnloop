<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| generated HTML -> browser | Static HTML/CSS/JS runs under `file://`; required content must be present before JS executes. |
| brandbook document -> local logo assets | Relative `../logo/*` images/downloads must resolve without remote requests or copied asset drift. |
| visual states -> readers | Status, contrast, theme, do/don't, and selected states must not rely on color alone. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-51-05 | Denial of Service | `brandbook/index.html` required content | mitigate | Generate essential content into committed HTML and forbid runtime fetch for required sections. |
| T-51-06 | Spoofing | Logo and usage section | mitigate | Render committed Phase 49 assets exactly through image/download links; do not redraw, recolor, outline-convert, or recompose logo assets. |
| T-51-07 | Tampering | Token and contrast presentation | mitigate | Read token values from generated `tokens.css`/`swatches.json` and badge classifications from Phase 48 evidence; do not store a new contrast matrix in `swatches.json`. |
| T-51-08 | Information Disclosure | Remote dependencies | mitigate | CSS/HTML source checks reject remote URLs, imports, iframes, beacons, and root-relative assets. |
| T-51-09 | Denial of Service | Responsive layout | mitigate | Responsive table wrappers and fixed logo/specimen dimensions prevent clipped or overlapping content across mobile/tablet/desktop. |
| T-51-SC | Tampering | npm/pip/cargo installs | accept | No new package installs are planned; existing local dependencies only. |
</threat_model>
