## Phase UI Boundary

Phase 48 does not add a new user-facing screen. It evolves the existing visual system by changing token values once in the canonical source and propagating those values to documented derivatives.

| Surface | Contract |
|---------|----------|
| Canonical source | `priv/static/cairnloop.css` `:root` and `[data-theme="dark"]` are the source of truth |
| Derivative 1 | `examples/cairnloop_example/assets/css/app.css` `@theme` and `@layer base` mirrors must match canonical values for every expressed `--cl-*` token |
| Derivative 2 | `prompts/cairnloop.tokens.json` must match canonical primitive and semantic values for its expressed keys |
| Prior proof | `logo/_contest/direction-boards.html` remains evidence only; do not edit it for Phase 48 unless a planning artifact explicitly requires a note update |
| Future surfaces | Production logo assets, favicon, OG image, README, and `brandbook/` are Phase 49-52 work, not Phase 48 |

Token evolution is additive and value-only: no existing `--cl-*` token may be renamed, removed, or repurposed in a way that breaks the sealed brand-token gate. New tokens are allowed only when they document a real semantic need and do not duplicate an existing token.

## Spacing Scale

Declared values use the existing Cairnloop scale from `priv/static/cairnloop.css` and remain multiples of 4 for UI specification purposes:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, focus offset, tiny proof labels |
| sm | 8px | Compact row gaps and inline metadata |
| md | 16px | Default panel padding and page gutter |
| lg | 24px | Section padding and stacked form groups |
| xl | 32px | Layout gaps and stat grouping |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Spacing debt: existing dense admin half-step tokens outside the declared Phase 48 spacing scale may remain only as pre-existing implementation debt. Phase 48 must not use, add, rename, document as accepted, or expand those values in new UI work. Minimum interactive target remains governed by `--cl-control-h-lg`, which is a control-height token and not part of the spacing scale.

## Color

Use a 60 / 30 / 10 Cairnloop color proportion across the existing operator UI after token propagation.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F4EEE2` light canvas, `#101614` dark canvas | Page backgrounds, broad shell surfaces |
| Secondary (30%) | `#FAF5EB` light surface, `#141B19` dark surface, `#1F2C28` raised dark surface | Cards, nav, panels, inputs, table surfaces |
| Accent (10%) | `#A8492A` light copper, `#D98A4A` dark copper | Primary actions, focus ring, active route marker, selected state edge, future C3.6 ring |
| Destructive | `#B54C36` light danger, `#C96A55` dark danger | Destructive/error states only; must pass button text contrast |

Accent reserved for: primary buttons, active route markers, focus rings, selected-state borders, the future C3.6 crowning loop ring, and small route-marker annotations. Do not use copper for every link, every heading, decorative fills, or non-selected panel chrome.

Required accessibility outcomes:

| Requirement | Contract |
|-------------|----------|
| Normal text | Every meaningful text pairing in `46-CONTRAST-BASELINE.md` must meet 4.5:1 after evolution |
| Large text / UI indicators | Large text, focus rings, route markers, and meaningful UI component boundaries must meet 3.0:1 |
| Known failures | Fix dark danger button text contrast, light muted-on-sunken text, and light neutral chip text |
| Fragile pass | Increase margin for light `--cl-text-muted` on `--cl-bg`; it must no longer sit at the prior 4.52:1 edge |
| Borders | Classify each failing border row as decorative or meaningful. Meaningful input/button boundaries must be changed to pass 3.0:1 |
| Color meaning | Status, error, selected, and warning states must continue to pair color with text or icon; never color alone |

The evolved palette must keep the brand read: calm basalt, warm paper, copper route marker, lichen success, waypoint-blue information, heather AI/eval, ember warning, and clay danger. Avoid pushing the UI into a one-note brown/orange, beige-only, dark-slate-only, or purple-blue palette.

## Token Propagation Contract

| Token class | Required action |
|-------------|-----------------|
| Primitive colors | Update canonical values first; mirror to `@theme --color-cl-*`, `@layer base --cl-color-*`, and `prompts/cairnloop.tokens.json color.primitive` |
| Light semantics | Resolve from updated primitives where possible; literal values must be mirrored in `app.css` and `tokens.json semantic_light` when those derivatives express the key |
| Dark semantics | Keep dark overrides semantic-only; mirror all expressed dark keys in `app.css` and `tokens.json semantic_dark` |
| Status triads | Tune surface / border / text values as needed for AA text contrast; document decorative border classification if border contrast remains below 3.0 |
| Focus ring | Keep copper-visible on light and dark surfaces; must pass non-text 3.0 against common surfaces |
| Typography tokens | Preserve names and stack values; derivative JSON must continue to express the same three stack roles |
| Shadow/radius/spacing scales | Do not redesign scales. Correct the known `--cl-shadow-raised` derivative drift so app.css no longer carries a divergent literal shadow value |

The derivative files must not become independent design sources. If a value exists in canonical and is expressed in a derivative, the derivative value must match canonical exactly after Phase 48.

## Interaction Contract

Phase 48 changes the look and accessibility of existing interactions through tokens only. It must not alter component behavior.

| Interaction | Contract |
|-------------|----------|
| Primary actions | Continue using copper surfaces with readable `--cl-primary-text`; hover uses `--cl-primary-hover` with AA text contrast |
| Danger actions | Light and dark danger buttons must both pass 4.5:1 for their text pairing |
| Focus | Existing focus ring remains visible on canvas, surface, raised, and sunken backgrounds |
| Navigation active state | Active routes keep a text label plus copper indicator; copper is not the only state cue |
| Status chips | Chips continue to pair status text with color; chip borders may be decorative only if the text/icon carries meaning |
| Theme | Light and dark token sets must remain coherent under `[data-theme="dark"]`; no JS theme behavior changes |
| Motion | No motion changes. Existing transform/opacity-only and `prefers-reduced-motion` contracts remain in force |

Do not add selection controls, token playgrounds, light/dark toggles, or brand-book interactive surfaces in this phase. Those belong to Phase 51 if needed.

## Verification Expectations

The planner/executor should verify the UI contract with source, diff, and rendered checks:

| Check | Required evidence |
|-------|-------------------|
| Canonical-first edit | `priv/static/cairnloop.css` is updated before derivatives; no existing `--cl-*` token is renamed or removed |
| Derivative parity | A diff/check proves `app.css` and `cairnloop.tokens.json` expressed values match canonical exactly |
| Contrast re-check | The full `46-CONTRAST-BASELINE.md` matrix is recomputed against evolved values; all meaningful text and UI pairings pass required thresholds |
| Failure closure | Dark danger, muted-on-sunken, neutral chip text, and fragile muted-on-bg outcomes are explicitly resolved |
| Border classification | Each low-contrast border row is marked decorative or changed to pass 3.0 when it is a meaningful UI boundary |
| Gate suite | `mix test`, `mix test.integration`, and `mix test.e2e` are green |
| Scope guard | `git diff --name-only` is confined to Phase 48-owned token/provenance/planning files; no production logo, README, favicon, OG, or brandbook changes |

Rendered-behavior checks that need a browser must be automated with the existing Playwright E2E path, not human UAT.
