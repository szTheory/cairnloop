| Property | Value |

## Phase UI Boundary

Phase 50 creates the scaffold and token derivation substrate only. It must not assemble the full brand book content that belongs to Phase 51.

| Surface | Contract |
|---------|----------|
| `brandbook/index.html` | One static shell page that proves local loading, relative assets, visual hierarchy, and token availability |
| `brandbook/assets/css/tokens.css` | Derived from canonical `priv/static/cairnloop.css` `:root` and `[data-theme="dark"]`; not manually redesigned |
| `brandbook/assets/css/brandbook.css` | Brandbook-only layout and presentation CSS that consumes tokens from `tokens.css` |
| `brandbook/TOKENS.md` | Provenance and regeneration note explaining how to update tokens without forking the palette |
| `brandbook/color/swatches.json` | Structured swatch data derived from canonical tokens or documented as a generated mirror |
| `brandbook/logo/` | Relative-copy destination or placeholder directory for Phase 51 logo gallery inputs |
| `brandbook/raster/` | Relative-copy destination or placeholder directory for allowed favicon/OG raster inputs only |

Do not wire `brandbook/` into the package, Phoenix endpoint, README, or example app. `mix.exs` package files remain `lib priv guides mix.exs README.md LICENSE CHANGELOG.md`; `brandbook/` stays git-tracked but unshipped.

## Scaffold Visual Contract

The Phase 50 scaffold should look like the first page of a professional brand reference, while remaining intentionally incomplete.

| Area | Required treatment |
|------|--------------------|
| Page shell | Full-width paper canvas using `--cl-bg`; content max-width `1120px`; page gutter `16px` mobile, `32px` desktop |
| Header | Compact brandbook title row with `Cairnloop brand book` and a provenance line, not a marketing hero |
| Status strip | Three compact status cells: `Self-contained`, `Canonical tokens`, `Ready for Phase 51` |
| Token preview | Small proof section showing live color chips and type samples pulled through CSS variables |
| Structure preview | Plain list of expected folders/files so Phase 51 has visible scaffolding to fill |
| Footer | Provenance note naming `priv/static/cairnloop.css :root` as source of truth |

The compact title/provenance header is the primary visual anchor, followed by the status strip and token preview.

Do not use oversized landing-page hero patterns, decorative gradient/orb backgrounds, nested cards, dark blurred imagery, or generic stock visuals. The phase output is a working reference shell, not a launch page.

## Spacing Scale

Declared values use the approved Cairnloop 4px rhythm from `priv/static/cairnloop.css`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline token/value gaps, chip internal gaps |
| sm | 8px | Compact rows, metadata spacing, swatch label gaps |
| md | 16px | Default block spacing, mobile page gutter, card padding |
| lg | 24px | Section padding, header/footer breathing room |
| xl | 32px | Desktop grid gaps, section-to-section rhythm |
| 2xl | 48px | Major vertical breaks between scaffold areas |
| 3xl | 64px | Top/bottom page padding on wide screens |

Exceptions: minimum interactive targets are `44px` for links/buttons/toggle controls if any are present. Existing canonical half-step tokens (`2px`, `12px`, `20px`, `40px`) may appear in copied token output because `tokens.css` is derived, but new brandbook layout CSS must use the declared scale above unless matching an existing `--cl-control-*` height.

## Typography

Use exactly these four sizes and two weights for the Phase 50 scaffold.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Metadata / token code | 12px | 400 | 1.5 |
| Body | 15px | 400 | 1.6 |
| Section heading | 18px | 600 | 1.44 |
| Page title / display proof | 28px | 600 | 1.29 |

Typeface usage:

| Typeface | Usage |
|----------|-------|
| Atkinson Hyperlegible Next / Atkinson Hyperlegible / system sans fallback | Body, navigation, labels, status text |
| Fraunces / Georgia fallback | Page title and display specimen only |
| Martian Mono / Atkinson Hyperlegible Mono / system mono fallback | Token names, hex values, file paths, derivation snippets |

No network font loading, font binaries, CDN calls, negative letter spacing, viewport-scaled type, or live text recreation of the logo wordmark is allowed. Fonts must degrade gracefully through the committed fallback stacks.

## Color

Use the evolved Refined palette already locked in canonical tokens. `brandbook/assets/css/tokens.css` must expose these values via the same `--cl-*` names, not a parallel brandbook palette.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F4EEE2` light canvas, `#101614` dark canvas | Page background and broad brandbook canvas |
| Secondary (30%) | `#FAF5EB` light surface, `#FFFFFF` light raised surface, `#141B19` dark surface, `#1F2C28` dark raised surface | Panels, token preview blocks, tables, status cells |
| Accent (10%) | `#A8492A` light copper, `#D98A4A` dark copper | Primary provenance marker, focus ring, selected/current marker, C3.6 ring reference, small route-marker accents |
| Destructive | `#B54C36` light danger, `#C96A55` dark danger | Error states and destructive warnings only |

Accent reserved for: provenance marker, focus ring, selected/current state borders, the C3.6 crowning-loop ring reference, primary action if a single control is present, and small route-marker annotations. Do not use copper for every heading, every link, broad backgrounds, decorative fills, table borders, or non-selected panel chrome.

All meaningful text pairings must meet WCAG AA 4.5:1. Meaningful focus indicators, swatch boundaries, and selected-state indicators must meet 3.0:1. Status/error/selected states must always pair color with text; never use color alone. Phase 48 reverified the core contrast rows and should be treated as the baseline.

## Token Derivation Contract

Phase 50 must make provenance visible and enforceable.

| Item | Contract |
|------|----------|
| Canonical source | `priv/static/cairnloop.css` `:root` and `[data-theme="dark"]` are the source of truth |
| Derived CSS | `brandbook/assets/css/tokens.css` preserves `--cl-*` token names and values from canonical output |
| Brandbook CSS | `brandbook/assets/css/brandbook.css` may add brandbook layout classes, but must consume `--cl-*` tokens rather than hard-code palette values |
| Swatch JSON | `brandbook/color/swatches.json` mirrors canonical primitive/semantic colors with token name, hex value, role, and theme where relevant |
| Provenance doc | `brandbook/TOKENS.md` names the source file, derivation method, regeneration command or manual mirror procedure, and drift-check expectation |
| No fourth palette | Do not create new palette literals beyond derived tokens, swatch JSON output, and documented verification examples |

The executor may use a small script, Mix task, shell pipeline, or documented manual extraction if it is repeatable and reviewable. The result must let a future maintainer regenerate or verify `tokens.css` without guessing.

## Interaction Contract

Phase 50 interactions are intentionally minimal because the full light/dark toggle and content sections belong to Phase 51.

| Interaction | Contract |
|-------------|----------|
| Local load | `index.html` opens from `file://` with no console errors and no failed network requests |
| Navigation | Same-page anchor links are allowed; all `href` values must be relative or fragment-only |
| Asset paths | CSS, JSON, logo, and raster references must use relative paths; no root-relative `/...` URLs |
| Theme | The scaffold may include non-interactive light/dark proof blocks using `[data-theme="dark"]`; do not ship a JS toggle unless Phase 50 planning explicitly includes it |
| Focus | Every focusable link/control has a visible token-driven focus ring using `--cl-focus-ring` or equivalent derived variables |
| Motion | No animation required. If hover/focus transitions are used, limit to color, border-color, background-color, opacity, or transform and honor `prefers-reduced-motion` |
| Downloads | Asset links can point to future-relative paths but should not pretend missing assets are downloadable |

No scripts are required for Phase 50. If a script is added, it must be local, optional, and pass the same file/network checks; external analytics, beacons, CDNs, imports, and remote images are prohibited.
