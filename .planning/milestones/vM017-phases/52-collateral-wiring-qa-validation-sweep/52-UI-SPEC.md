## Phase UI Boundary

Phase 52 owns the final real-world collateral placements and the QA/report states that prove they work.

| Surface | Contract |
|---------|----------|
| `README.md` | Lead with the approved horizontal SVG logo using a repo-relative path that GitHub can render, then keep badges and install/docs content intact |
| `examples/cairnloop_example/priv/static/images/logo.svg` | Replace the placeholder logo with the approved Cairnloop public lockup or intentionally copied approved mark/lockup SVG |
| `examples/cairnloop_example/priv/static/favicon.ico` | Replace with the approved `logo/favicon.ico` output |
| `examples/cairnloop_example/priv/static/images/` | Add only approved copied static assets needed by root metadata, such as `cairnloop-og.png` and optional favicon PNG/SVG copies |
| `examples/cairnloop_example/lib/cairnloop_example_web/components/layouts/root.html.heex` | Add favicon and OG/social metadata with Phoenix static paths and accessible document title semantics |
| `examples/cairnloop_example/test/e2e/*` | Add a gated Playwright E2E proving the logo and favicon/metadata render through the app |
| QA evidence | Record SVG validity, raster budget, package hygiene, diff scope, and test results in phase summary/validation artifacts |

Out of scope: changing selected C3.6 logo geometry, editing canonical token values, rebuilding the brandbook content, adding a Phoenix route for `brandbook/`, changing `mix.exs` package files, adding remote assets, or adding npm/package UI dependencies.

Owner logo-family sign-off remains the subjective precondition before wiring. Once wiring starts, implementation and verification are automated; do not create human UAT tasks for rendered logo/favicon behavior.

## Collateral Placement Contract

| Placement | Required visual treatment |
|-----------|---------------------------|
| README header | First visible line is the approved logo image, centered or left-aligned simply, with alt text `Cairnloop`; the existing text H1 may be removed or demoted so the SVG is the brand header |
| README badges | Badges remain below the logo/header with existing links; remote shields may remain because they are pre-existing README badges, not new brand imagery |
| Example app static logo | Use the committed Cairnloop SVG asset, not a recreated live-text wordmark; transparent background; no rectangular cage; no subtitle |
| Browser favicon | Use approved simplified favicon assets from `logo/`; do not scale the full lockup to favicon sizes |
| OG/social image | Use the approved `cairnloop-og.png` raster for `og:image`; keep the 1200x630 image unchanged |
| App root metadata | Add `og:title`, `og:description`, `og:image`, `og:type`, and favicon links using local static paths; do not add analytics or remote preview services |

The default public logo remains `logo/cairnloop-lockup-horizontal.svg`. The icon-only mark is allowed only where the wordmark is too small or redundant. The tagline lockup is not allowed for README header, favicon, nav, package identity, or app root metadata.

## Spacing Scale

Declared values use the approved Cairnloop 4px rhythm. Phase 52 should mostly preserve existing spacing; new README/app logo placement CSS or test fixtures must use these values when spacing is needed.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline logo/badge gaps, focus offset |
| sm | 8px | Compact metadata rows and badge spacing |
| md | 16px | README logo-to-badge spacing, default asset proof spacing |
| lg | 24px | Header breathing room and QA report section spacing |
| xl | 32px | Desktop logo/header separation when needed |
| 2xl | 48px | Major QA/report breaks |
| 3xl | 64px | Page-level spacing only if a rendered proof page is created |

Exceptions: fixed proof and asset sizes from prior phases remain authoritative: favicon `16px` and `32px`; icon mark minimum `24px`; horizontal lockup minimum `112px` width; OG image `1200x630`; focusable target minimum `44px` for any new control/link in rendered proof or app UI.

## Typography

Use exactly these four UI sizes and two weights for any new app-facing labels, README-adjacent HTML proof, or QA output UI. The committed SVG wordmark is not live text and is exempt from UI type sizing.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Metadata / file label | 12px | 400 | 1.5 |
| Body | 15px | 400 | 1.6 |
| Section heading | 18px | 600 | 1.44 |
| Page title / proof heading | 28px | 600 | 1.29 |

Typeface usage:

| Typeface | Usage |
|----------|-------|
| Atkinson Hyperlegible Next / Atkinson Hyperlegible / system sans fallback | Example app UI, README prose, QA labels, status text |
| Fraunces / committed vector wordmark | Only the approved logo SVG wordmark; do not recreate it with live text |
| Martian Mono / Atkinson Hyperlegible Mono / system mono fallback | File paths, token names, command output, raster sizes, QA report values |

No new font files, CDN fonts, negative letter spacing, viewport-scaled type, or live-text logo recreation is allowed.

## Color

Use the evolved Refined palette already locked in canonical tokens and committed logo assets.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F4EEE2` light canvas, `#101614` dark canvas | Existing app/page backgrounds and README/brand proof surfaces |
| Secondary (30%) | `#FAF5EB` light surface, `#FFFFFF` light raised surface, `#141B19` dark surface, `#1F2C28` dark raised surface | Existing app shell surfaces, metadata proof surfaces, QA tables |
| Accent (10%) | `#A8492A` light copper, `#D98A4A` dark copper | Committed logo ring, focus ring, active route marker, selected/current QA badge, one primary action if a proof page needs it |
| Destructive | `#B54C36` light danger, `#C96A55` dark danger | Failed QA/error states only; no destructive product action in Phase 52 |

Accent reserved for: the C3.6 crowning-loop ring in logo assets, focus rings, active route markers, selected/current QA badges, and small route-marker annotations. Do not recolor logo assets to make broader decorative use of copper. Do not use copper for every link, heading, table border, or broad background.

All meaningful text pairings must meet WCAG AA 4.5:1. Meaningful focus indicators and selected/current indicators must meet 3.0:1. Status/error/selected states must pair color with visible text such as `PASS`, `FAIL`, `Current`, or `Blocked`; never use color alone.

## Asset Semantics And Accessibility

| Asset/state | Contract |
|-------------|----------|
| README logo alt | `Cairnloop` |
| Example app logo accessible name | Use `alt="Cairnloop"` for `<img>` usage or SVG `role="img"` plus accessible title/label if embedded |
| Decorative duplicates | If the same logo appears redundantly near text `Cairnloop`, mark only the duplicate decorative, not the sole brand identifier |
| Favicon | Browser icon only; no alt text; verify network status and dimensions through browser/source checks |
| OG image | Metadata image only; use `og:image:alt` with `Cairnloop — Support that leaves a trail.` when supported by the root layout |
| SVG hygiene | Every committed SVG has `xmlns`, valid `viewBox`, no external refs, no embedded raster, no script/foreignObject, and no editor metadata |

The README logo must remain GitHub-renderable: use a repo-relative path such as `logo/cairnloop-lockup-horizontal.svg`, not `file://`, root-relative `/...`, raw GitHub CDN URLs, or base64 data.

## Interaction Contract

Phase 52 adds no new product workflow. It validates existing rendered behavior after collateral wiring.

| Interaction | Contract |
|-------------|----------|
| App load | Example app pages render with the new logo static asset available and without console/page/request failures caused by collateral |
| Favicon/meta | Browser receives favicon links and OG metadata from `root.html.heex`; assets resolve through local Phoenix static paths |
| Logo display | The new logo renders without distortion, clipping, rectangular cage, or low-contrast arbitrary background |
| Links | README and any local proof links remain keyboard-focusable and repo-relative where possible |
| Motion | No new motion. Existing transform/opacity-only and `prefers-reduced-motion` contracts remain in force |
| Failure handling | QA output must name the failed asset/path and next action; no vague `asset failed` messages |

Rendered-behavior checks that need a browser must be automated with `PhoenixTest.Playwright.Case` and `@moduletag :e2e`. Use `evaluate/3` for `getBoundingClientRect()`, `document.querySelector(...)`, computed asset URLs, favicon link discovery, and image natural-size checks as needed.
