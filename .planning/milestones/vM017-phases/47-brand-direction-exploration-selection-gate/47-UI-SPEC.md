## Phase UI Boundary

The UI artifact is `logo/_contest/direction-boards.html`. It may use inline CSS/SVG or same-folder relative assets under `logo/_contest/assets/`. It must not depend on a server, network request, package install, build step, CDN font, Phoenix route, or npm tool.

This phase formalizes an already-complete owner selection, not a new selection workflow. The board must visibly mark:

| Gate item | Locked selection |
|-----------|------------------|
| Logo | C3.6 crowning-loop cairn: open copper ring is the top stone; wide flat stones; compact ring |
| Palette | Refined basalt/trailpaper/copper tuning |
| Type | Current stack: Atkinson Hyperlegible + Fraunces + Martian Mono |

Out of scope: editing `priv/static/cairnloop.css`, `examples/cairnloop_example/assets/css/app.css`, `prompts/cairnloop.tokens.json`, README, example-app logo/favicon, `brandbook/`, `mix.exs`, or production logo assets.

## Layout Contract

Use one static page with these ordered sections:

| Section | Contract |
|---------|----------|
| Header | Title `Cairnloop direction boards`; subdued status line `Selection recorded: C3.6 / Refined / current type`; no marketing hero |
| Selection summary | Three compact cells for logo, palette, and type; selected state uses copper edge + text label, never color alone |
| Direction roster | Four direction panels: A stacked-cairn/wrapping loop, B negative-space loop, C integrated typemark, D waymark/contour glyph; C3.6 is shown as the selected refinement |
| Proof rows | For every direction, render 16px, 24px, 48px, and 256px marks; include horizontal and vertical lockups; include light and dark surfaces |
| No-cage proof | Explicit row showing marks on transparent/checker or outlined host surfaces without a rectangular logo cage |
| Palette/type board | Current, Conservative, Refined, and Bolder palette chips may appear as historical context; Refined is selected. Fraunces is selected; Spectral is rejected/context only |
| Notes | Short durable notes: selection already made in `47-DISCUSSION-LOG.md`; exact token propagation occurs in Phase 48; production assets occur in Phase 49 |

Responsive behavior: single-column at narrow widths, two-column direction grid at >= 768px, four-column proof strips only when they fit without horizontal text clipping. SVG proof sizes are fixed and must not stretch: 16/24/48/256 CSS pixels.

## Color

Use the selected Refined palette in the contest board as preview values only. Do not mutate canonical tokens in Phase 47.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F4EEE2` light, `#141B19` dark | Page canvas, large proof backgrounds |
| Secondary (30%) | `#FAF5EB` light surface, `#1E2A24` dark surface | Direction panels, proof cells, summary cells |
| Accent (10%) | `#A8492A` light copper, `#D98A4A` dark copper | Selected C3.6 ring, selected-state edge, focus ring, selected chip label, route-marker proof only |
| Destructive | `#C96A55` | Rejected labels or contrast-failure warnings only; no destructive actions in this phase |

Accent reserved for: the selected crowning loop ring, selected-state borders/markers, focus rings, selected palette/type chips, and small route-marker annotations. Do not use copper as every link, every heading, or general decoration.

Neutral preview colors: selected stone/base `#141B19`, mid stone `#1E2A24` on light, dark-mode stones `#F4EEE2` and `#B7C0B2`, muted text `#5E665D`.

Accessibility constraints:

| Constraint | Contract |
|------------|----------|
| Contrast | Board text pairings must meet WCAG AA: 4.5:1 for normal text, 3.0:1 for large text and non-text UI indicators |
| Color meaning | Every selected/rejected/status state must pair color with text label such as `Selected`, `Rejected`, `Preview`, or `Phase 48` |
| Phase 46 carry-forward | Refined palette copy must state that Phase 48 must resolve the Phase 46 text failures, fragile muted pairing, dark danger failure, and classify border failures |
| No-cage | No full-canvas logo background rectangles inside SVG marks; surface preview cells may have backgrounds outside the transparent SVG to prove contrast |

## Logo And SVG Contract

All direction marks must be hand-authored SVG with `xmlns`, `viewBox`, accessible name, and transparent mark background.

Selected C3.6 reference geometry:

| Element | Geometry |
|---------|----------|
| Ring | `circle cx="24" cy="15" r="5.4"`, `fill="none"`, `stroke-width="2.8"` |
| Mid stone | `rect x="12" y="25" width="24" height="7" rx="3.5"` |
| Base stone | `rect x="7" y="34" width="34" height="8" rx="4"` |

Forbidden in SVGs: `<image>`, external `href`, `xlink:href`, `data:` payloads, embedded raster, editor metadata, JavaScript, animation, full-canvas background rect cages, chat-bubble silhouette, infinity-symbol read, or an icon-left-of-plain-text treatment presented as the selected direction.

The integrated `oo` typemark must appear only as an explored/rejected direction or note. The selected wordmark is plain `cairnloop` in Fraunces.

## Interaction Contract

This is a static evidence page, not an app workflow.

| Interaction | Contract |
|-------------|----------|
| Selection | Non-interactive display of the locked owner choice; no voting, scoring, chooser UI, or active form |
| Links | Optional relative links to `47-DISCUSSION-LOG.md` or source docs may be plain anchors; they must remain keyboard focusable |
| Theme | Show light and dark proof surfaces simultaneously; do not require a toggle |
| Motion | None required. If hover/focus transitions are used, limit to opacity/transform/color over <= 180ms and respect `prefers-reduced-motion` |
| Failure handling | If an asset fails to render, visible fallback text must identify the direction and mark it as a rendering problem |
