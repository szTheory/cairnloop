  <action>
    Create `.github/workflows/release.yml` to handle automated publishing to Hex.pm and hexdocs for future tags.
    
    Structure the workflow (D-03, D-04, D-12):
    - `on: push: tags: ["v*"]`
    - Top-level `env: ACTIONS_RUNNER_NODE_VERSION: "24"` (to match `ci.yml` and meet the June 2 deadline).
    - A single job `publish` that runs on `ubuntu-latest`.
    - Add `if: github.event_name == 'push' && startsWith(github.ref, 'refs/tags/v')` to the job.
    
    Steps:
    1. Checkout code (`actions/checkout@v4`).
    2. Setup Elixir (`erlef/setup-beam@v1` with `elixir-version: "1.19.0"` and `otp-version: "27.2"`).
    3. Restore cache (`actions/cache@v4` with paths `deps`, `_build` and key keyed on `mix.lock`).
    4. Install dependencies (`mix deps.get`).
    5. Publish package: run `mix hex.publish --yes`.
    6. Publish docs: run `mix hex.publish docs --yes`.
    
    Provide the Hex API key to the publish steps via the `env:` block in those specific steps or the job block: `HEX_API_KEY: ${{ secrets.HEX_API_KEY }}`.
  </action>
  <verify>
    <automated>ls .github/workflows/release.yml</automated>
  </verify>
  <done>The `release.yml` file is created and matches the requirements for automated Hex publishing.</done>
