WidgetChannel (lib/cairnloop/channels/widget_channel.ex):
- `join("widget:lobby", _payload, socket)` reads `socket.assigns.user_token`, calls `Chat.create_customer_conversation(%{host_user_id: token})`, assigns `:conversation_id`, and returns `{:ok, %{conversation_id: id}, socket}`. (`create_customer_conversation/1` broadcasts `{:conversations_changed}` on the `"conversations"` topic.)
- `handle_in("new_message", %{"content" => content}, socket)` reads `conversation_id = socket.assigns[:conversation_id]` (T-M001: NEVER from payload), enqueues `ProcessMessage.new(%{channel: "widget", conversation_id: conversation_id, content: content})` via `Oban.insert/1`, returns `{:reply, :ok, socket}`.
- subscribe_and_join pattern:
  `{:ok, reply, socket} = socket(Cairnloop.Channels.WidgetSocket, "widget_socket:demo_customer", %{user_token: "demo_customer"}) |> subscribe_and_join(Cairnloop.Channels.WidgetChannel, "widget:lobby", %{})`. The join reply is `%{conversation_id: id}`.

ProcessMessage worker (lib/cairnloop/workers/process_message.ex) — widget branch (D-09, direct perform/1):
- `ProcessMessage.perform(%Oban.Job{args: %{"channel" => "widget", "conversation_id" => conversation_id, "content" => content}})` → `:ok` on success. `conversation_id` is the INTEGER id from the join reply. The worker calls `Chat.ingest_widget_message/2`, which inserts the customer Message and broadcasts `{:conversations_changed}` on `"conversations"` + `{:message_created, message.id}` on `"conversation:#{conversation_id}"`.

InboxLive PubSub (lib/cairnloop/web/inbox_live.ex):
- Subscribes to `"conversations"` on `Cairnloop.PubSub` inside the `connected?` block (mount).
- `handle_info({:conversations_changed}, socket)` reloads `Chat.list_conversations()` and re-assigns. After ingest, a fresh `render(inbox_view)` reflects the new conversation.

Chat facade (lib/cairnloop/chat.ex): `create_customer_conversation/1` and `ingest_widget_message/2` both broadcast `{:conversations_changed}` on `"conversations"`.
