## Spacing Scale

Declared values (this project uses a 2/4px rhythm for dense admin UI; standard 8pt multiples hold for layout):

| Token | Value | Usage in This Phase |
|-------|-------|---------------------|
| `--cl-space-1` | 2px | Row micro-gap (e.g. eyebrow to headline) |
| `--cl-space-2` | 4px | Icon gap, inline label/value gap |
| `--cl-space-3` | 8px | Vertical rhythm between fields; `cl-stack` default gap |
| `--cl-space-4` | 12px | Section-internal padding; button group gap |
| `--cl-space-5` / `--cl-space-gutter` | 16px | Card body padding (`.evidence-rail > .cl-card`) |
| `--cl-space-7` | 24px | Rail card gap (`.evidence-rail gap`) |
| `--cl-space-8` | 32px | Conversation layout column gap (stacked mode) |

Exceptions:
- **44px minimum tap target** on the density toggle button and Expand-all/Collapse-all controls — use `--cl-control-h-lg: 44px`.
- **`data-density="compact"` modifier** reduces card padding from `--cl-space-gutter` (16px) to `--cl-space-4` (12px) and rail-card gap from `--cl-space-7` (24px) to `--cl-space-4` (12px). Density affects spacing only — never open/close state.

> Source: codebase `cairnloop.css:86-101`, CONTEXT.md D-04 (density = spacing only), ROADMAP.md Phase 43 (≥44px tap targets — apply here proactively for controls added this phase).

| Warning | `--cl-warning` / `--cl-warning-text` | `#8B531E` / `#7A4818` | `#D98A4A` / `#E8B488` | High-risk tier chip; budget-risk signals |
| AI/eval | `--cl-ai` / `--cl-ai-text` | `#7A5D78` / `#5F4A5D` | `#C9A7C6` / `#D6BCD3` | Trace ID accents; AI-surface source cards |

**Accent `--cl-primary` is reserved exclusively for:**
1. The 2px left route-marker on the active/selected `cl_card` (`.cl-route-active`)
2. Primary button background (`cl_button variant="primary"` — Approve action)
3. Focus rings (`--cl-focus-ring`)
4. The `summary` hover underline when an operator interacts with a `cl_disclosure`

Never use `--cl-primary` for section labels, body text, or generic borders.

**State must always pair color + text + (where space allows) icon** — never color alone. All chips (`cl_chip`) emit a variant icon SVG alongside the label. The risk tier chip uses text ("Risk: High") and color class together.

## Interaction Contract

This section is required for this phase because it is primarily an interaction-restructuring phase with no new visual components being introduced.

### Tier Structure (RAIL-01, RAIL-02)

```
governed-action-card
├── [TIER 1 — ALWAYS VISIBLE, NEVER IN <details>]
│   ├── Eyebrow (GOVERNED ACTION)
│   ├── Headline (snapshotted title/consequence)
│   ├── Status chip (cl_chip — text + icon + color)
│   ├── Meta row: risk tier chip + approval mode (text)
│   ├── Approval outlook sub-line (if non-nil)
│   ├── Block reason copy (if scope_invalid / policy_denied)
│   └── Footer: Approve / Reject / Defer (if active_approval.status == :pending)
│
├── [TIER 2 — cl_disclosure, phx-update="ignore", data-tier="2"]
│   ├── "Inputs & scope" group
│   │   ├── Humanized input rows (cl_fact_list)
│   │   └── [TIER 3 nested] cl_disclosure "Raw input snapshot" (default closed)
│   ├── "History" group
│   │   ├── Event mini-timeline items
│   │   └── [TIER 3 nested per-event] cl_disclosure "Details" (conditional, default closed)
│   └── "Policy explanation" group
│       ├── Calm policy sentence
│       └── [TIER 3 nested] cl_disclosure "Raw policy snapshot" (default closed, if policy_snapshot non-nil)
│
└── [TIER 3 standalone — cl_disclosure, phx-update="ignore", NO data-tier="2"]
    └── "Identifiers & trace" group (D-02; default closed)
        └── cl_fact_list: Proposal · Tool · Version · Idempotency key
```

**Invariant:** The pending footer (`active_approval.status == :pending`) must appear in Tier 1, never inside any `<details>` element. Expand-all/Collapse-all cannot hide the approval decision surface.

### cl_disclosure Usage

All `<details>` elements in the rail use `cl_disclosure/1` (Phase 37 primitive):
- `id` attribute: stable, unique per-card (e.g. `"ga-#{proposal.id}-inputs"`, `"ga-#{proposal.id}-history"`, `"ga-#{proposal.id}-policy"`, `"ga-#{proposal.id}-trace"`)
- `phx-update="ignore"`: prevents PubSub re-renders from snapping panels shut (RAIL-02 invariant)
- `open` attribute: static HTML boolean at initial render only — never a live server assign
- `:summary` slot: text label at 13px/400 weight in `--cl-info` color (inherits `.cl-details > summary` CSS)
- Tier-2 groups carry `data-tier="2"` on the `<details>` element for JS scoping

### Auto-Expand (D-08, RAIL-03)

Initial `open={true}` is set on the **Inputs & scope** `cl_disclosure` when:
- `active_approval.status == :pending`, OR
- `block_reason` is `scope_invalid` or `policy_denied`

Additionally, when `block_reason == :policy_denied`, the **Policy explanation** `cl_disclosure` also renders with `open={true}`.

Auto-expand is initial-render-only. A card that becomes pending mid-session via PubSub will NOT auto-snap open — this is the deliberate invariant cost (D-09, locked).

### Rail-Level Controls (RAIL-03)

A **rail control bar** renders at the top of the `.evidence-rail` (above the first governed-action card):

```
[Expand all]  [Collapse all]  [Comfortable / Compact]
```

- **Expand all / Collapse all**: `Phoenix.LiveView.JS` `set_attribute({"open", ""})` / `remove_attribute("open")` targeting `[data-tier="2"]` within the rail scope. Does NOT touch Tier-1 content or Tier-3 (`data-tier` absent) elements.
- **Density toggle**: a small JS hook reads/writes `localStorage` key `cl:rail:density`; applies `data-density="comfortable"` (default) or `data-density="compact"` on the `.evidence-rail` container on mount.
- Controls use `--cl-control-h-sm: 28px` (ghost button variant) to stay visually subordinate; minimum touch target 44px via padding.
- Placement: single row, right-aligned within the rail header, before the first card.

### Density CSS Contract

New CSS rules in `cairnloop.css` (no new tokens required):

```css
/* Comfortable (default) — existing values */
.evidence-rail[data-density="comfortable"] > .cl-card {
  padding: var(--cl-space-gutter, 16px);
}
.evidence-rail[data-density="comfortable"] {
  gap: var(--cl-space-7, 24px);
}

/* Compact */
.evidence-rail[data-density="compact"] > .cl-card {
  padding: var(--cl-space-4, 12px);
}
.evidence-rail[data-density="compact"] {
  gap: var(--cl-space-4, 12px);
}
```

Default is `comfortable` (matches existing CSS). CSS attribute selector runs without JS — no flash of wrong density if hook fires after paint.

## Copywriting Contract

Brand voice: calm, action-oriented, reason-forward, never raw terms to operators (§5.2 Admin UI tone). State = text + color + icon, never color alone (§7.5).

| Element | Copy |
|---------|------|
| Primary CTA (pending approval) | **Approve** |
| Secondary CTA (pending approval) | **Reject** (requires reason) |
| Tertiary CTA (pending approval) | **Defer** (requires reason) |
| Reject reason placeholder | `Reason for rejection (required)` |
| Defer reason placeholder | `Reason for deferral (required)` |
| Approval required label | `Approval required` |
| Reason required note | `A reason is required for rejection or deferral.` |
| Tier-2 summary — Inputs & scope | `Inputs & scope` |
| Tier-2 summary — History | `History` |
| Tier-2 summary — Policy explanation | `Policy explanation` |
| Tier-3 summary — Raw input snapshot | `Raw input snapshot` |
| Tier-3 summary — Per-event details | `Details` |
| Tier-3 summary — Raw policy snapshot | `Raw policy snapshot` |
| Tier-3 summary — Identifiers (D-02) | `Identifiers & trace` |
| Empty history state | `No history yet.` |
| Rail control — expand | `Expand all` |
| Rail control — collapse | `Collapse all` |
| Density toggle — comfortable | `Comfortable` |
| Density toggle — compact | `Compact` |
| Risk tier chip prefix | `Risk:` (e.g. `Risk: High`) — text + color, never color alone |
| Approval mode label | `Approval:` (e.g. `Approval: Required`) |
| Block reason — scope invalid | `This action was blocked: scope validation failed.` |
| Block reason — policy denied | `This action was blocked by policy.` |

**Destructive actions in this phase:**
- **Reject**: irreversible approval decision — confirmed by required reason textarea (`rows="2"`, `required` attribute, no modal confirm needed; the required field is the friction gate).
- **Defer**: soft hold — confirmed by required reason textarea (same pattern as Reject).
- No hard delete actions exist in this phase.

**Error states:**
- Reject/defer form submitted with empty reason: browser-native validation (`required` attr) — no custom error copy needed at this phase.
- PubSub disconnect while panel is open: panel stays open (no server re-close); no user-facing message needed.

> Source: brand book §5.2, §5.5, §13.2; CONTEXT.md (Tier-2/3 group names from D-01/D-02 specifics); existing `conversation_live.ex:1083-1106`.
