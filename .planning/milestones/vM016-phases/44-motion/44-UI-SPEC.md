## Motion Contract

This section is the primary deliverable for Phase 44. All other design dimensions (typography,
color, spacing) are unchanged from the shipped system — only motion is new.

### Token Reference (existing — do not add new duration values)

| Token | Value | Purpose |
|-------|-------|---------|
| `--cl-dur-instant` | 100ms | Press feedback, toggles |
| `--cl-dur-micro` | 140ms | Tooltips, hover, icon swap |
| `--cl-dur-ui` | 180ms | Gate state-flip; UI state changes |
| `--cl-dur-panel` | 260ms | Drawer / rail / source-card reveal |
| `--cl-dur-exit` | 160ms | Dismissals — faster than entrance |
| `--cl-dur-route` | 600ms | Deliberate route-line / marker motif (v2 AMOTION only) |
| `--cl-ease-out` | `cubic-bezier(0.23, 1, 0.32, 1)` | Default entrance |
| `--cl-ease-in-out` | `cubic-bezier(0.77, 0, 0.175, 1)` | Directional / symmetrical |
| `--cl-ease-drawer` | `cubic-bezier(0.32, 0.72, 0, 1)` | Rail / panel / drawer reveal |
| `--cl-ease-linear` | `linear` | Fade-only (opacity transitions) |
| `--cl-stagger` | 50ms | Per-item delay step for list entrance |

Source: `cairnloop.css:142-153` (already shipped). Rationale for no new tokens: brand book §15.1
bands are fully covered — <180ms UI state, 240–320ms panel reveal, 160ms exit faster than entrance.

### Mechanism

- **Pure CSS only.** `@keyframes` + `animation` and CSS `transition` on utility classes.
- **`phx-mounted={JS.transition(...)}`** for one-shot mount entrances (hero count, toast enter).
  This is an existing repo pattern (`search_modal_component.ex:65`). No new JS hooks. No WAAPI.
- **All motion CSS under `.cl-app`** so the global `prefers-reduced-motion` block
  (`cairnloop.css:197-211`) zeroes it automatically.
- **Two-file mirror:** new rules go into `priv/static/cairnloop.css` (lib canonical) AND
  `examples/cairnloop_example/priv/static/assets/css/app.css`.

Source: CONTEXT.md D-01, D-02.

### Surface-by-Surface Motion Specification

#### 1. Hero Count Entrance

| Property | Value |
|----------|-------|
| Element | `.cl-hero__count` (`.cl-stat__count` for secondary stats) |
| Animation | `@keyframes cl-enter-up`: `opacity: 0 → 1`, `translateY(4px) → translateY(0)` |
| Duration | `--cl-dur-ui` (180ms) — must be **< 180ms** per MOTION-01 |
| Easing | `--cl-ease-out` |
| Trigger | `phx-mounted={JS.transition("cl-motion-enter", ...)}` one-shot mount |
| Replay | **None on count ticks.** The `.cl-hero__count` text node carries **no `transition-property`**. morphdom patches the text node in place without re-inserting the element, so the `@keyframes` does not replay on value updates. |
| Forbidden | No count-up / number-roll tween. No `width`/`height`/`top`/`left`/`max-height` in this animation. |

Class: `.cl-motion-enter` (utility that applies the `cl-enter-up` animation one-shot).

#### 2. Rail / Drawer Reveal

| Property | Value |
|----------|-------|
| Element | `.cl-drawer`, `.evidence-rail` (and any future rail container) |
| Animation | CSS `transition: transform var(--cl-dur-panel) var(--cl-ease-drawer), opacity var(--cl-dur-panel) var(--cl-ease-drawer)` |
| Transform | `translateX(16px) → translateX(0)` for side-entry (right rail); `translateX(-16px) → translateX(0)` for left-entry |
| Duration | `--cl-dur-panel` (260ms) |
| Easing | `--cl-ease-drawer` |
| Forbidden | No `width`, `max-width`, `height`, `max-height`, `top`, `left` transitions. |

Class: `.cl-motion-reveal` — sets the initial hidden state + transition; paired with a presence class (e.g. `data-open="true"` or LiveView JS toggle) to drive the transition.

#### 3. Gate State-Flip

| Property | Value |
|----------|-------|
| Element | Policy-gate state cells (pending → draft-only / blocked / allowed transitions) |
| Mechanism | **Reuse existing `.cl-motion-state`** (already in `cairnloop.css:207-210`) |
| Transition | `transition-property: opacity, color, background-color, border-color` |
| Duration | `--cl-dur-ui` (180ms) |
| Reduced-motion | **Preserved at 120ms** — this is the meaning-bearing cross-fade that survives the global `prefers-reduced-motion` zeroing (`cairnloop.css:207-210`). It is the only motion that survives reduced-motion. |
| Constraint | Never state-by-color-alone (brand §7.5) — the textual/iconic state label must change alongside the color cross-fade. |

No new CSS needed for this surface — `.cl-motion-state` is already shipped. Phase work is confirming the gate state cells carry the class.

#### 4. List-Item Enter (Staggered)

| Property | Value |
|----------|-------|
| Element | `<li>` rows in inbox list (`.cl-list-row` inside `inbox_live.ex`) |
| Animation | `@keyframes cl-enter-up` (same as hero) — `opacity: 0 → 1`, `translateY(4px) → translateY(0)` |
| Duration | `--cl-dur-ui` (180ms) per item |
| Stagger | `nth-child(-n+5)` delays: item 1 = 0ms, item 2 = 50ms, item 3 = 100ms, item 4 = 150ms, item 5 = 200ms. Items 6+ have no delay (effectively instant, no stagger cap animation needed). |
| Step | `--cl-stagger` (50ms) |
| Trigger | CSS `animation` fires on DOM insert. morphdom does not replay on attribute-only patches (bulk-select, filter, count change). |
| Insert-only invariant | **Stagger plays on first paint and on real new-row insert only.** Returning to the screen via LiveView navigation must NOT re-stagger existing rows. Implementation guard: a `phx-mounted` one-shot that adds the stagger class once, or scoping the stagger animation class so it does not apply to already-settled elements. Lightest mechanism wins — do not over-engineer. (Planner decision from D-05.) |
| Forbidden | No replay on mere navigation/arrival. No `width`/`height` animation. |

Class: `.cl-list-stagger` on the `<ul>` container (or equivalent) with `nth-child` delay rules on direct `<li>` children.

#### 5. Toast / Flash Enter and Exit

| Property | Value |
|----------|-------|
| Element | Flash/toast container (see placement decision below) |
| Enter animation | `@keyframes cl-toast-enter`: `opacity: 0 → 1`, `translateY(-8px) → translateY(0)` (slides down from top-right) |
| Enter duration | `--cl-dur-ui` (180ms) |
| Enter easing | `--cl-ease-out` |
| Exit animation | `@keyframes cl-toast-exit`: `opacity: 1 → 0`, `translateY(0) → translateY(-4px)` |
| Exit duration | `--cl-dur-exit` (160ms) — **faster than entrance** |
| Exit easing | `--cl-ease-in-out` |
| Trigger | `phx-mounted={JS.transition("cl-toast-enter", ...)}` on mount; `phx-remove={JS.transition("cl-toast-exit", ...)}` on dismiss |

**Toast placement decision (D-08):** The lib layer (`Cairnloop.Web.Components`) does not currently define a `cl_flash`/`cl_toast` component — flash rendering lives in the example app's `core_components.ex` (DaisyUI-based). Phase 44 adds a lib-layer `cl_flash/1` component with the brand-token `.cl-toast` CSS surface (`background: var(--cl-surface-raised)`, `border: 1px solid var(--cl-border)`, `box-shadow: var(--cl-shadow-modal)`, `border-radius: var(--cl-radius-md)`). This is consistent with the component-elevation posture and makes the motion treatment reusable across any host app. The example app's DaisyUI-based `flash/1` can coexist independently.

#### Motion Not Applied (Negative Contract)

The following elements and events must produce **zero CSS transitions** on their triggering elements. This is a success criterion (SC-3) and must be tested.

| Surface | Reason |
|---------|--------|
| Reply-send button + payload | No transition — avoids implying AI "thinking" |
| `⌘K` / search modal open | No transition on the triggering key or search input |
| Hero count value ticks (live PubSub updates) | Text node patched in place; no `transition-property` on `.cl-hero__count` text |
| Secondary stat count ticks | Same as hero count |
| Any `width`, `height`, `top`, `left`, `max-height` transition | Absolutely forbidden in all new motion rules |

### Forbidden Motion Motifs (brand §15.3)

- Typing indicator for AI
- Robot-thinking animation
- Sparkle explosions
- Infinite pulsing around AI cards
- Confetti for resolved support conversations
- Route-line draw / marker-travel / source-card stack reveal / FLIP list reorder (deferred to v2 as AMOTION-01 / AMOTION-02; these require `phx-hook` + WAAPI)

### Reduced Motion (MOTION-02)

The global `@media (prefers-reduced-motion: reduce)` block at `cairnloop.css:197-211` zeroes all
animation and transition durations under `.cl-app` to `0.01ms`. This block is already shipped.

Phase 44 work is:
1. Ensure every new entrance animation sits under `.cl-app` (so the global block zeroes it).
2. Confirm `.cl-motion-state` is the **only** motion that survives — the re-enable rule at
   `cairnloop.css:207-210` explicitly restores it to 120ms cross-fade.
3. No JS toggle is needed — the media query honors the OS preference live.

Result under reduced-motion: all transform/translate animations are removed; the gate state-flip
cross-fade (opacity/color/background-color/border-color) is kept at 120ms as the comprehension aid.

## Copywriting Contract

Phase 44 is a pure CSS motion phase. It introduces no new user-facing copy surfaces except the
`cl_flash/1` component (new lib-layer toast).

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable — Phase 44 adds no new primary action buttons |
| Empty state | Not applicable — Phase 44 does not add new empty states |
| Error state | Not applicable — Phase 44 does not add new error surfaces |
| Toast (info kind) | Inherits operator-facing copy from the calling LiveView's `put_flash` — e.g. "Conversation resolved." (calm, past-tense, operator-grade) |
| Toast (error kind) | Inherits from `put_flash` — e.g. "Could not deliver reply. Draft is intact." (brand §5.6 shape: what happened, why, what next) |
| Flash dismiss | `aria-label="Dismiss"` on the close button (accessible, terse) |

Copywriting rules in effect (no change from established system):
- Calm, plain, action-oriented (brand §5.2 "Admin UI" tone).
- Past-tense for confirmations; present-tense for status.
- No emoji in operator-facing flash messages.
- Never apologize on behalf of the user; apologize when the system failed (brand §5.3 rule 9).
