## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| stored JSONB snapshot → Preview live leg | `input_snapshot` rehydrated to call host `preview/1`; attacker-influenced snapshot keys/values cross here. |
| host tool `preview/1` → LiveView render | third-party host code returns prose that becomes operator-visible. |
| snapshot maps → operator UI (presenter) | raw stored values (incl. PII in `input_snapshot`, raw `inspect`'d `policy_snapshot.reason`) flow toward display. |
| Phase 14 trust fields → Phase 15 approval surface | risk that Phase 15 reuses the live prose leg instead of D-16 snapshotted columns. |
