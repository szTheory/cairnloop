<objective>
Harden the first write path to be low-surprise under replay and worker retries (M011-S04-03), and
make execution telemetry bounded and observability-only (OBS-01). This means: completing the
three-layer at-most-once defense (Oban uniqueness + pre-execution terminal guard + deterministic
run-level idempotency key passed into `run/3` via context), wiring transient-vs-terminal retry
semantics with per-attempt events, extending `Cairnloop.Governance.Telemetry` with two enum-bounded
execution events, emitting them AFTER the co-commit, and proving all of it (DB-backed for replay
guarantees, headless for telemetry-bounding and branch logic).

Purpose: This is the milestone's first side effect — it must never double-write, never leak
high-cardinality payload into metric labels, and degrade gracefully under retries.

Output: Extended bounded telemetry module; run-level key derivation + retry semantics finalized in
the worker; full DB-backed at-most-once/retry/idempotency proof and headless telemetry-bounding proof.
</objective>
