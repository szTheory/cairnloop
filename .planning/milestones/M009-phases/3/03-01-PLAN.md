  <action>
Update `Cairnloop.Web.SearchModalComponent` to act as a hidden-by-default global search modal.

**Analog Reference:** Follow the analog `lib/cairnloop/web/conversation_live.ex` defined in `3-PATTERNS.md` for the UI/Core Component pattern. 
Core UI Pattern excerpt:
```elixir
    <div class={["rail-card host-context", @error && "error"]}>
      <h3>Customer Context</h3>
      <%= if @error do %>
        <p>Customer context is unavailable right now. Continue handling the conversation, then reload to try again.</p>
      <% else %>
```

Key implementation details:
- **State**: The component must manage `:show` (boolean), `:query` (string), and `:results` (list).
- **Shortcut Interception**: On the outer-most visible element (or a hidden top-level div if unshown), use `phx-window-keydown="toggle_search"` and `phx-key="k"` with `phx-target={@myself}` to intercept keydown events efficiently.
- **Event Handler**: In `handle_event("toggle_search", params, socket)`, check if `params["metaKey"]` or `params["ctrlKey"]` is true. If so, toggle the `:show` assign.
- **Immediate Focus**: Use `phx-mounted={JS.focus()}` on the `<input>` element so it receives focus immediately when rendered. Add `phx-debounce="300"`.
- **Search Querying**: Handle the `phx-change="search"` event.
  - If `byte_size(query) < 3`, clear results.
  - Otherwise, send a `Req.post` to Scrypath following the pattern in `3-PATTERNS.md`: `Application.get_env(:cairnloop, :scrypath_api_url, "https://api.scrypath.local/v1/index")` + `/search` (assuming `/search` endpoint for querying). Pass `json: %{query: query}`. Update `:results` with the response.
- **Error Handling**: Use the "Flash Error Handling" shared pattern from `3-PATTERNS.md` if the query fails:
  ```elixir
  {:noreply, put_flash(socket, :error, "Search service unavailable. Please try again later.")}
  ```
- **UI Design**: Follow `03-UI-SPEC.md`. 
  - Backdrop: subtle overlay, high z-index (`z-50`, `fixed`).
  - Container padding: 16px (`md`).
  - Placeholder: "Search resolved conversations (cmd+k)..."
  - Display "No results found." and "Try adjusting your search terms..." for empty states if the query is filled but returns nothing.
  - Error state: "Search service unavailable. Please try again later."
