<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Application write paths -> Oban jobs | Publish and resolve operations enqueue durable indexing work |
| Canonical KB corpus -> Assistive resolved-case corpus | Separate storage must preserve truth hierarchy |
| Retrieval storage -> future callers | Stored evidence must not leak tenant/visibility context |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-M009-S01-01 | Tampering | Retrieval migrations and chunk schemas | mitigate | Enforce unique natural keys for replacement writes so retries cannot silently duplicate corpus rows |
| T-M009-S01-02 | Information Disclosure | Resolved-case evidence tables | mitigate | Store assistive evidence separately and keep bounded metadata only; do not treat raw transcripts as the searchable document |
| T-M009-S01-03 | Denial of Service | Indexing workers | mitigate | Keep heavy chunking/embedding work in Oban and preserve idempotent retry-safe transactions |
</threat_model>
