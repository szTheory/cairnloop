## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Retrieval callers -> retrieval facade | All future UI and AI consumers must go through one safe API |
| Provider queries -> result ranking | Filtering must happen before fusion |
| Developer recovery commands -> corpus jobs | Bulk replay/rebuild paths must stay explicit and bounded |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-M009-S01-04 | Information Disclosure | Provider query layer | mitigate | Apply tenant, audience, and visibility predicates inside candidate queries before ranking or merging |
| T-M009-S01-05 | Tampering | Result normalization | mitigate | Return explicit `source_type` and `trust_level` so assistive evidence cannot masquerade as canonical KB truth |
| T-M009-S01-06 | Denial of Service | Replay and rebuild commands | mitigate | Require explicit scope parameters and route work back through Oban instead of synchronous bulk rebuilds |
</threat_model>
