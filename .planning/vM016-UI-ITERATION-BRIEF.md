## Context — why this milestone

The operator dashboard shipped a real design system in 0.4.0 (tokens + components + Cockpit Home +
nav). The owner wants to take it to the next level: consistent, branded, intuitive, maximally useful
per persona/JTBD, reaping compounding dividends from a tightened design system — focusing **especially
on areas not yet heavily iterated**. The owner asked directly: *what is the user journey / persona /
JTBD, and how should the screens thread together?* — because the UI "feels disjoint."

**Reframe that drives everything:** this is **NOT** "build the foundation." Three parallel Explore
agents confirmed the foundation is already strong — **~137 design tokens** across every scale (color,
spacing, type, control sizing, radius, shadow, z-index, **motion incl. Emil Kowalski easing**), a
**BEM + `.cl-` utility** architecture (no Tailwind, no build step — one shipped
`priv/static/cairnloop.css`), a **token-pure component library**, a **brand-token gate**, and a
**Playwright screenshot pipeline**. The real work: **(a) harvest the dividends** (componentize
repeating patterns), **(b) stop the drift** (complex screens leaked off-system), **(c) fix the IA**
(orientation, primacy, threading), **(d) make every screen fully express its states** (seed),
**(e) lock the gains in** (harden the gate). Owner's chosen scope: **comprehensive sweep.**

## The persona / JTBD / IA narrative (the answer to "what's the main thing?")

**5 jobs, but not 5 co-equal personas.** Primary user = **solo founder / small technical team; one
person wears every hat in a day.**

| Job (Home verb) | Persona | Screen |
|---|---|---|
| **Work the queue** (PRIMARY) | front-line operator | Inbox → Conversation |
| Recover resolved | governance/recovery | Inbox (resolved) |
| Tend knowledge | KB editor | Knowledge Base (gaps→suggestions→editor) |
| System health | SRE/ops | Settings |
| Audit trail | auditor | Audit Log |

**Honest hierarchy:** jobs 2–5 are **consequences of job 1**. Resolved-recovery, gaps, and the audit
trail are all *generated by working the queue*; system-health is an orthogonal *check*, not a job.
The current Home pretends all five are equal — which is *why it feels disjoint*. The IA should say:
"there is one main thing — working the queue — plus things you dip into."

**Threading broken today (verified):** "Recover resolved" CTA goes to `/inbox` **without the resolved
filter** (broken on click); **no "next in queue"** after a conversation; **Audit log is a dead-end
leaf** (rows don't link to their subject); **`cl_breadcrumb` exists but is wired nowhere**; every
screen hand-rolls its own header/width so they read like different apps (the outer `.cl-main` shell
is consistent — the *inner* frame drifts).

## Current-state assessment (evidence-grounded)

**Strong:** token system (all scales); BEM+`.cl-` architecture; token-pure `components.ex`
(`cl_button/cl_card/cl_chip/cl_banner/cl_empty/cl_stat/cl_shell/cl_breadcrumb/cl_icon`); brand gate;
screenshot pipeline (`examples/cairnloop_example/screenshots/capture.mjs` → `guides/assets/`, 17 shots).

**Weak spots (the targets):**
1. **Design-system DRIFT in bespoke complex screens.** `conversation_live.ex` ~15 off-palette hex
   (browns `#8b7355`/`#4c4033`, maroon `#8b1a1a`, cream `#fef9e5`/olive `#7a5c00`, gray `#e5e7eb`,
   `#fffdf8`, bronze `#c38f57`) in approve/reject/defer + KB-fix + raw-expander markup; ~68 inline
   `style=` across `lib/cairnloop/web/`; `search_modal_component.ex` *returns* inline rgba/hex from
   `*_style/1` helpers. Components are pure; the bespoke markup leaked.
2. **Gate hole** — catches only `var(--cl-*, #hex)` fallbacks, not inline `style="…#hex…"`, raw
   `rgba()`, or helper-returned hex. Drift re-enters silently.
3. **IA/orientation** — Home egalitarian; inconsistent count-color semantics; "Degraded" status in a
   count slot; dead 6th grid cell; no cross-job threading; inconsistent page-shells.
4. **Seed doesn't fully express the UI** — audit rows share one timestamp + reasons all "—"; only 1 KB
   suggestion (filter tabs → empty); no rejected/deferred/published suggestions; no MCP tokens (token
   list/mask UI never shown); no draft-state article; dark mode never captured; only InternalNote tool.
5. **Motion defined but unused.**
6. **Latent bugs found:** `cl-gap-2`/`cl-align-center`/`cl-justify-between` referenced in markup but
   **never defined in CSS** (inert); Recover-resolved CTA filter; `.cl-table` has no overflow wrapper
   (breaks in narrow / host-constrained containers).

## Decided directions (researched; sources cited in research outputs)

### D1 — Home: two-tier primacy model
**Hero "Work the queue"** (full-width, ~2–3× weight): single **copper** count + real `cl_button`
primary CTA; **Recover-resolved folds in** as a quiet sub-line (same `/inbox` surface) *with the
resolved filter applied*, omitted when zero. **Secondary band "Tend the trail"** (3-up, calmer):
Tend knowledge / Audit / System health, with **neutral (not copper) counts** (copper = route marker;
protects 70/20/10), and **System health as a `cl_chip`** (success "Healthy" / **warning** "Degraded"
— Ember, *not* Fault Clay). **Zero state** → calm Lichen success (icon+text, never confetti).
*Implementation:* add `cl_hero` (or `cl_stat variant="hero"` + `:detail` slot); **de-polymorphize
`cl_stat` to numeric-only** (the `count :any` accepting a number-or-health-string is the semantics
bug); route health through `cl_chip`; keep `safe/2` fail-closed counts; **footgun:** `assign_counts/1`
re-queries everything per PubSub tick → use count queries + throttle. *Why:* answers "what's the main
thing?" honestly; matches operator-tool consensus (Linear/Help Scout/Oban Web lead with the work);
egalitarian grids = "vanity dashboard" antipattern.

### D2 — Conversation rail: safety-pinned accordion
**Reorder rail so decisions come first.** **Tier 1 (never collapses):** headline + status + the
**safety quartet** (risk tier · confidence/grounding · policy outcome · approval mode) + the pending
**Approve/Reject/Defer** footer. **Tier 2 (≤3 collapsible `<details>` groups/card):** Inputs&scope ·
History (last-event summary in the `<summary>`) · Policy explanation. **Tier 3 (nested `<details>`):**
raw snapshots, per-event metadata, trace ids. *Mechanism:* **native `<details>`/`<summary>`** for all
per-card disclosure (idiomatic, a11y-free, SSR/patch-safe, **no server assigns** — the 4 PubSub reload
handlers would fight assigns-based open state and snap panels shut); **`Phoenix.LiveView.JS` only** for
rail-level **Expand-all/Collapse-all** + a **remembered density toggle** (localStorage hook — Option
C's instinct, layered on, never touching Tier 1). **Auto-expand blocking/pending cards.** Extract a
shared `cl_disclosure`/`cl_details` primitive. *Why:* brand "make safety visible" + NN/g "never hide
safety-critical info" both place the disclosure line *below* the safety signals; Sentry's issue-detail
redesign + Stripe disputes converged here. *Footguns:* never wrap a Tier-1 signal (incl. policy
*outcome*) in `<details>`; stable `id` per `<details>`, don't bind `open` to assigns; don't collapse
the pending decision; cap ~3 Tier-2 groups; audit `#8b7355` summary contrast (WCAG AA).

### D3 — Responsive: desktop-first cockpit, authored mobile-first, tablet-complete, phone-graceful
**Methodology = mobile-first `min-width`** (convert the two `max-width:640` blocks). **Optimization
target = desktop** (every B2B operator tool invests there; phones get native apps — Zendesk/Front/
Intercom; Oban Web/LiveDashboard are desktop+scroll). **Breakpoints:** standardize **640 / 768 (new
tablet) / 1024**, **documented as literal constants in one CSS comment block.** **Footgun (captured):**
`var()` is illegal in `@media`/`@container` conditions — **do NOT tokenize breakpoints as custom
properties** (silent no-op); add real *layout* tokens instead (`--cl-content-max`, `--cl-rail-width`,
`--cl-page-gutter`). **Non-optional fix:** wrap every `.cl-table` in an accessible `overflow-x:auto`
scroller (`role=region`, `tabindex=0`, aria-label). **In scope:** mobile-first normalization, 768
tablet, table scrollers, verify conversation 2-col stacks below `lg` + rail wraps, sticky bulk-bar
wraps/clears last row, ≥44px tap targets, responsive-relevant inline styles → classes. **Deferred:**
phone-optimized patterns (tabbed Timeline↔Detail, card-transform tables, off-canvas nav, container
queries — sticky hazard + arbitrary host containers make viewport media queries the v1 choice).

### Design-system dividends (cross-cutting)
- **Componentize:** `cl_page` (shell: title/subtitle/`:actions`/`:breadcrumb`/`:subnav` slots +
  `:wide`/`:reading` width), `cl_fact_list` (labeled-facts — highest reuse), `cl_source_card`
  (evidence/source — deletes search-modal `*_style` helpers), `cl_status_cell` (audit chips via
  `AuditLogPresenter.action_tone/1`), `cl_switch` (real `role=switch`), `cl_disclosure`.
- **Drift map:** `#e5e7eb`→`--cl-border`; `#8b7355`→`--cl-text-muted`; `#4c4033`→`--cl-text`;
  `#fffdf8`→`--cl-primary-text`; maroon `#8b1a1a`+`#fdecea`→`--cl-danger-*`; cream/olive
  `#fef9e5`/`#7a5c00`→`--cl-warning-*`; bronze `#c38f57`→`--cl-border` (neutral input); search-modal
  `#4A6238`→success / `#3F6F80`→info via `cl_source_card source_variant`. Replace hand-rolled
  approve/reject/defer footer with `cl_button` variants + `cl-textarea`. Migrate inline layout →
  `.cl-` utilities; **define the inert `cl-gap-2`/`cl-align-center`/`cl-justify-between`**.
- **Drift-proofing (the dividend-protector):** harden the brand-token gate to fail on inline
  `style="…#hex…"`, raw `rgba()/hsl()`, helper-returned hex in render `.ex` files; anchor on `#` +
  color-context to avoid false positives; magic-comment allowlist; CSS file stays unscanned. Add a
  complementary Credo check (dev-time); ExUnit test remains CI truth.
- **Motion (restrained, brand §15, reduced-motion-safe):** hero count opacity+translate <180ms; rail/
  drawer reveal 260ms `--cl-ease-drawer`; **gate state-flip** 180ms tagged `.cl-motion-state`; list
  enter opacity+translateY(4px) staggered ≤5; toast enter/exit; route-line motif 600ms. **Never:**
  reply-send, every keystroke (⌘K opacity-only/none), count ticks, infinite pulse, sparkle/elastic,
  layout-property animation.

### Seed enrichment
Audit: varied timestamps (~14d) + real reasons + a **rejected** and a **deferred** event + action
variety. KB suggestions: **rejected / deferred / published** examples. Settings: 1–2 **masked MCP
tokens**. KB: a **draft-state article**. Governed actions: a **second tool type / higher risk tier**.
**Dark-mode capture** context for the 3–4 highest-signal screens.

## Phase decomposition (each independently shippable; phases 37+)

37. **Component primitives** (additive): `cl_page`, `cl_hero`/`cl_stat` split, `cl_disclosure`,
    `cl_fact_list`, `cl_source_card`, `cl_status_cell`, `cl_switch` + CSS; layout tokens; define the
    inert utilities + `.cl-table` scroll-wrapper.
38. **Shared page-shell migration** — Home/Inbox/Audit/Settings/KB → `cl_page`; wire `cl_breadcrumb`
    on the deep KB-from-conversation path.
39. **Home primacy redesign (D1)** — two-tier hero + band; health-as-chip; count-color fix; dead cell;
    zero-state; count queries + throttle; fix Recover-resolved filter CTA.
40. **Drift remediation + gate hardening** (paired — no regression window) — adopt new components in
    `conversation_live` + `search_modal`; hex→token map; inline-layout→utilities; harden gate + Credo.
41. **Conversation rail progressive disclosure (D2)** — reorder; Tier 1/2/3 native-`<details>`;
    auto-expand blocking; rail expand-all + density toggle.
42. **Cross-screen threading** — next-in-queue; audit-row↔conversation; governed-action↔audit;
    article→originating-conversation.
43. **Responsive (D3)** — mobile-first normalization; 768 tablet; table scrollers; stacking/bulk-bar/
    tap-target verification.
44. **Motion** — apply restrained plan; reduced-motion verified live.
45. **Seed enrichment + screenshot regen + verification sweep** — enrich states; regenerate light+dark
    screenshots; per-screen acceptance; `mix test` green (hardened gate + `golden_path_test.exs`).

*Ordering:* primitives → shell → Home (answers the headline question early) → drift+gate (quality
spine) → rail (biggest UX lift, benefits from `cl_page`/`:reading`) → threading → responsive → motion
→ seed+verify.

## Verification loop

- **CI truth = `test/integration/golden_path_test.exs`** (Phoenix.LiveViewTest; deterministic; no
  browser). `mix test` excludes `:integration` locally — validate with
  `PGPORT=5432 PGUSER=postgres PGPASSWORD=postgres MIX_ENV=test mix test.integration` + the quality
  lane before claiming green. (Postgres live on :5432; app default 5433 → always pass `PGPORT=5432`.)
- **Screenshot pipeline = per-screen visual acceptance** (non-gating): boot seeded demo, capture, diff
  the touched screens, **light + dark** (dark mode is the tell for drift remediation — off-palette hex
  didn't theme, tokens do). Motion is zeroed in capture → verify motion **live**; confirm static
  end-states unchanged.
- **The hardened brand-token gate (phase 40)** is what makes the consistency gains *stick*.

## Carried guardrails (do not violate)

- CSS = **BEM + `.cl-` utilities**, one shipped `cairnloop.css`, **no Tailwind / no build step**.
  Tokens (`--cl-*`) are the single source of truth; **no hardcoded hex in render `.ex`**.
- Brand: copper = route marker not landscape (70/20/10); **never status-by-color-alone**; raw terms
  only behind expanders; calm/fail-closed/reason-forward copy; Degraded=Ember, blocked/failed=Fault
  Clay; motion §15.
- Arch: durable Ecto+events = truth (telemetry = observability only); reads via `Cairnloop.Governance`
  facade; snapshot trust facts at decision time; seal completed paths (additive > rewrite).
- Ship via release-please PRs (feat:/fix: → release; auto-merge). Most phases are `feat:`/`fix:` →
  several minor/patch releases across the milestone.
