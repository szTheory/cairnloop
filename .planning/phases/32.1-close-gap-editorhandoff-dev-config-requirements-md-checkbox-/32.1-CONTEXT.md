## Implementation Decisions

### INT-BLOCKER-01: EditorHandoff secret_key_base config

- **D-01:** Add the following to `examples/cairnloop_example/config/dev.exs` (static dev-only
  key, min 64 bytes, clearly labeled dev-only):
  ```elixir
  config :cairnloop, Cairnloop.KnowledgeAutomation.EditorHandoff,
    secret_key_base: "dev_only_64_byte_minimum_secret_for_editor_handoff_tokens_cairnloop"
  ```
  Placement: after the existing `CairnloopExampleWeb.Endpoint` config block, before any
  `import_config` lines at the bottom.

- **D-02:** Add the following to `examples/cairnloop_example/config/runtime.exs` (fail-closed
  `fetch_env!` — raises at startup if env var absent, which is the right prod posture):
  ```elixir
  config :cairnloop, Cairnloop.KnowledgeAutomation.EditorHandoff,
    secret_key_base: System.fetch_env!("CAIRNLOOP_HANDOFF_SECRET_KEY_BASE")
  ```
  Placement: inside the production-only `if config_env() == :prod do` block (or equivalent
  conditional block) in `runtime.exs` — mirrors the existing pattern for other prod-only secrets.

- **D-03:** Add a new "EditorHandoff Token Key" section to `guides/04-troubleshooting.md`
  following the existing "Symptom / Cause / Fix" pattern:
  - **Symptom:** Clicking "Open for manual edit" in the KB suggestions list crashes with a
    500 error in dev, or the app fails to start in production with `RuntimeError: ...secret_key_base`.
  - **Cause:** `Cairnloop.KnowledgeAutomation.EditorHandoff` requires a `secret_key_base` in
    application config. In the `:test` environment a persistent_term fallback is used; all other
    environments require explicit config.
  - **Fix:** Add the config key (dev value shown for local development; use an env var in
    production — see exact snippets for both).
  Placement: add after the existing pgvector section (end of the guide), as its own `##` section.

### BRAND-04 gate fix

- **D-05:** In `test/cairnloop/web/brand_token_gate_test.exs`, update both wildcard calls
  from `*.ex` to `**/*.ex` for consistent recursive scanning:
  ```elixir
  # Before:
  Path.wildcard(Path.join(@web_dir, "*.ex")) ++
    Path.wildcard(Path.join(@example_live_dir, "*.ex"))

  # After:
  Path.wildcard(Path.join(@web_dir, "**/*.ex")) ++
    Path.wildcard(Path.join(@example_live_dir, "**/*.ex"))
  ```
  Both wildcards get the fix: `@web_dir` to catch `knowledge_base_live/` subdirectory files
  (the audited gap), and `@example_live_dir` for consistent future-proofing. The existing
  `refute files == []` guard ensures the test fails fast if path resolution breaks.
