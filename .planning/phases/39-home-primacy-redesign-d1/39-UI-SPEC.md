## Spacing Scale

Cairnloop uses a named-step scale (not raw 8pt multiples), with half-steps for dense admin UI. All values are declared as `--cl-space-*` tokens. Phase 39 uses the following:

| Token | Value | Usage in Phase 39 |
|-------|-------|-------------------|
| `--cl-space-2` | 4px | Icon gaps, chip icon gap, inline micro padding |
| `--cl-space-3` | 8px | Gap between hero elements (`cl-hero` flex gap), applied-filter row gap, chip gap |
| `--cl-space-4` | 12px | Secondary stat card inner padding (via `--cl-space-gutter` = 16px fallback) |
| `--cl-space-5` | 16px | `--cl-space-gutter`: panel padding; gap between hero and band; `.cl-home-grid` gap |
| `--cl-space-7` | 24px | Hero top/bottom padding (`cl-hero` uses `var(--cl-space-7) 0`) |
| `--cl-space-10` | 48px | `cl_empty` top/bottom padding (zero-state hero swap) |

**Scale source:** `cairnloop.css` lines 87–101. Pre-existing — not re-specified.

Exceptions:
- Touch targets: `--cl-control-h-lg` = 44px for the hero CTA (`cl_button` size `lg`); minimum 44px for all interactive tap targets (brand §16.2, WCAG 2.5.5).
- Applied-filter dismiss link uses `--cl-control-h-md` = 36px (non-critical secondary action, desktop-primary context).

## Color

Admin UI proportion target: **70% neutral surfaces / 20% text + borders / 10% semantic state** (brand §7.3). Phase 39 restructures Home to honor this split by reserving copper exclusively for the hero.

| Role | Token | Light value | Usage in Phase 39 |
|------|-------|-------------|-------------------|
| Dominant (70%) | `--cl-bg` | `#F5F0E6` (Trailpaper) | Page background (`cl_shell`, `cl_page`) |
| Secondary surfaces (20%) | `--cl-surface` | `#FBF7EE` (Warm Stone) | Secondary band stat cards (`.cl-stat` background), applied-filter row background |
| | `--cl-border` | `#D8D0BF` (Granite) | Card borders, dividers |
| | `--cl-text` | `#18211F` (Basalt) | Stat job labels, hero job label |
| | `--cl-text-muted` | `#677066` (Slate Lichen) | Stat meta, hero detail sub-line, applied-filter label text |
| Accent — copper (10%) | `--cl-primary` | `#A94F30` (Path Copper) | Hero count ONLY; primary CTA button fill; active nav marker |
| | `--cl-on-primary` | `#FFFFFF` | Text on copper CTA button |
| Semantic — success | `--cl-success` | `#4A6238` (Deep Lichen) | "All caught up" zero-state hero count tint (`cl-hero__count--calm`), "Healthy" chip border/text |
| | `--cl-success-surface` | `#EDF1E2` | "Healthy" chip background; applied-filter "Resolved" chip background |
| | `--cl-success-border` | `#C9D3A6` | "Healthy" and "Resolved" chip border |
| | `--cl-success-text` | `#3C5430` | Text inside success chips |
| Semantic — warning | `--cl-warning-surface` / `--cl-warning-border` / `--cl-warning-text` | see CSS lines 63–65 | "Degraded" health chip only |
| Semantic — neutral | `--cl-neutral-surface` | `#EFEADF` | "Count unavailable" quiet sub-line background (D-06 error state) |
| | `--cl-neutral-text` | `--cl-text-muted` | "Count unavailable" text |

**Accent reserved for:** (1) Hero copper count digit, (2) Primary "Open inbox" CTA button fill, (3) Active nav marker. Copper is NOT used on secondary-band stat tiles, filter chips, or health chip. Copper is the route marker, not the landscape (brand §7.3, D1 70/20/10 palette).

**Destructive:** `--cl-danger` / `--cl-danger-surface` — Phase 39 has no destructive actions; these tokens are not introduced here.

**Never state by color alone** (brand §7.5): all chips pair color + icon + text label; zero-state swaps icon + text, not color alone.

**Source:** `cairnloop.css` lines 37–67; `CONTEXT.md` D1 decision; brand book §7.3, §7.5. Pre-existing tokens — not re-specified.

## Layout Architecture

### Home: Two-tier structure (Phase 39 target)

```
<.cl_page width="wide">
  <!-- Tier 1: Primary hero (full-width) -->
  <.cl_hero job="Work the queue" count={@open_count} href="/inbox" …>
    <:detail>
      <!-- Recover-resolved sub-line: visible only when @resolved_count > 0 -->
      <.link href="/inbox?status=resolved">
        {@resolved_count} resolved — eligible for recovery
      </.link>
    </:detail>
    <:cta_slot>
      <.cl_button variant="primary" size="lg" …>Open inbox</.cl_button>
    </:cta_slot>
  </.cl_hero>
  <!-- Zero-state hero swap: rendered when @open_count == 0 in place of cl_hero -->
  <.cl_empty icon="check-circle" title="All caught up">
    Nothing is waiting on you right now.
  </.cl_empty>

  <!-- Tier 2: Secondary band (3-up grid) -->
  <div class="cl-home-grid cl-mt-5">
    <.cl_stat job="Tend knowledge" … />
    <.cl_stat job="Audit trail" … />
    <!-- Health cell: cl_stat shell but chip replaces count slot -->
    <div class="cl-stat">
      <span class="cl-stat__job">System health</span>
      <.cl_chip variant={@health_variant} label={@health_label} />
      <span class="cl-stat__meta">{@health_meta}</span>
    </div>
  </div>
</.cl_page>
```

**Grid:** `.cl-home-grid` — mobile-first 1→2→3 columns (`min-width: 640px`, `min-width: 1024px`). Pre-existing in `cairnloop.css` line 425–427. The dead 6th grid cell (previously the 5th `cl_stat` leaving a phantom empty cell at 3-up) is removed by using exactly 3 secondary tiles.

### InboxLive: applied-filter row

When `status=resolved` is active, render above the conversation list:

```
<div class="cl-applied-filter cl-row cl-text-small">
  <.cl_chip variant="success" label="Resolved" />
  Showing resolved conversations ·
  <.link patch={~p"/inbox"}>Show all</.link>
</div>
```

When the resolved list is empty, use `cl_empty` with body copy (see Copywriting).

## Interaction Contract

### Hero tier
- `cl_hero` renders as a `<section>`, not a link tile. The CTA is an explicit `cl_button variant="primary" size="lg"` inside `:cta_slot`.
- The Recover-resolved sub-line in `:detail` is a plain `<a href="/inbox?status=resolved">` — no `phx-click`, no JS. A simple href is correct: navigating to a URL is not a LiveView event.
- When `@open_count == 0`: replace the `cl_hero` block with `cl_empty`; the secondary band persists below (D-07). No whole-page celebration. No confetti.
- When count is unavailable (error path): show `0` as the integer count (fail-closed) AND a quiet `"Count unavailable"` neutral sub-line below the count (D-06).

### Secondary band
- Each secondary `cl_stat` tile is a clickable link tile (existing `cl_stat` renders `<.link navigate={@href}>`). Copper is not spent here.
- System health tile: uses the `.cl-stat` card shell (div, not `.link`) containing a `cl_chip` in the slot where the count number would appear. Health state conveyed by chip icon + label + variant — never color alone (brand §7.5).
- Secondary stat counts are neutral text (`--cl-text`), not copper. The `calm?` attribute is `false` for Tend/Audit counts (no calm tinting on secondary band) unless genuinely zero, in which case the subtle `--cl-success` tint on the count is acceptable.

### InboxLive filter wire-up (HOME-02 / D-01 through D-05)
- New `handle_params/3` in `InboxLive` reads `?status=resolved` (or nil).
- `normalize_status/1` whitelist: `"resolved"` → `:resolved`, anything else → `nil` (unfiltered). No `String.to_existing_atom` on raw input.
- PubSub re-query uses `@status` socket assign — filter-aware, cannot leak open conversations into the resolved view.
- `prune_selected_ids/2` called from both `handle_params/3` and `handle_info/2` to clear bulk selections that no longer match the filtered list.
- Applied-filter row renders a `cl_chip variant="success" label="Resolved"` + `Show all` patch link. Absent when no filter active.

### Count queries + throttle (HOME-05 / D-09)
- Home uses `Chat.count_conversations(status: :open)` and `Chat.count_conversations(status: :resolved)` — cheap `SELECT count(*)` queries, not `Enum.count` over a full list load.
- PubSub burst throttle: `Process.send_after(self(), :recount, 500)` + a `pending_recount?` flag in socket assigns to coalesce `{:conversations_changed}` bursts into at most one recount per ~500ms window.
- `safe/2` fail-closed helper preserved: count errors degrade to `0` (integer) + "Count unavailable" sub-line, never an exception.

### Focus and keyboard
- Hero CTA `cl_button` is keyboard-reachable and has the copper focus ring (`--cl-focus-ring`).
- Applied-filter "Show all" link is keyboard-reachable.
- No new modal dialogs or overlay layers in this phase.
