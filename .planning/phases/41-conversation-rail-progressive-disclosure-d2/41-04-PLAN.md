## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client JS hook → browser localStorage | The density hook reads/writes `localStorage["cl:rail:density"]`. Value is a controlled enum ("comfortable"/"compact"); it only sets a `data-density` attribute consumed by CSS attribute selectors — no markup injection. |
| rail JS commands → DOM `[data-tier='2']` | Expand/Collapse-all toggle the boolean `open` attribute on first-party `<details>` elements. No untrusted selector or value. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-41-05 | Tampering | density hook localStorage value → data-density attribute | mitigate | The hook applies the stored value only as a `data-density` attribute consumed by static CSS selectors; it never injects markup or evaluates the value. A tampered localStorage value yields at worst an unstyled attribute (no XSS, no privilege). |
| T-41-06 | Elevation/Tampering | Expand/Collapse-all JS DOM ops | accept | Pure client-side toggle of the boolean `open` on Tier-2 `<details>`; Tier-1 (the approval decision surface) is structurally outside any `<details>` and cannot be hidden (RAIL-01, verified Plan 03). No server state or authz is touched. |
| T-41-SC | Tampering | npm/pip/cargo installs | accept | No package installs; the colocated hook + app.js wiring use existing `phoenix_live_view` 1.1 colocated-hook machinery + existing `phoenix-colocated` build output. No new dependency (RESEARCH: not applicable). |
