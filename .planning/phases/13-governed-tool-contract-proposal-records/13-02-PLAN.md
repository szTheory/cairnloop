<objective>
Add the durable governance record layer and the public `Cairnloop.Governance` facade. Two schemas (`ToolProposal` + append-only `ToolActionEvent`) mirror the `ReviewTask`/`ReviewTaskEvent` idiom exactly (D-20). A migration creates both tables with an idempotency unique index and a cascade FK. `Governance.validate/3` is one pure, re-callable, ordered `with` pipeline whose clause order IS the fail-closed precedence (D-15, D-17); typed input is validated through the tool's `changeset/2` callback (D-04). `Governance.propose/3` is the thin persistence wrapper: it synchronously co-commits the proposal + a `proposal_created` event in a transaction, snapshots resolved tier/mode/policy at propose time, derives a deterministic idempotency key, returns the existing proposal on duplicate, and rejects unknown tools pre-persistence with telemetry only. `Governance.Policy.resolve/3` is the Phase 15 approval-resolver seam. Telemetry is observability only.

Purpose: This is the workflow-truth and fail-closed core of vM011 (TOOL-03, TOOL-04). The `validate/3` pipeline is the function the Phase 15 resume worker re-calls; the snapshots are what Phase 15 re-validates against; the reserved columns are the Phase 16 execution seam. Nothing executes this phase — even `:auto` tools only produce a durable proposal. Per the shift-left posture (D-31), pipeline internals, snapshot builders, idempotency derivation, and policy resolution are settled in this plan and hidden behind the narrow facade.
Output: 4 new modules under `lib/cairnloop/governance/`, the `Cairnloop.Governance` facade, a migration, and 3 new test files (Wave 0).
</objective>
