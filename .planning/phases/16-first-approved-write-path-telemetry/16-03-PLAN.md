<objective>
Reflect execution outcomes into the operator surface and prove OBS-02 attribution is reconstructable
from durable records. Map the new `:executed` / `:execution_failed` terminal statuses into the
existing four display groups with ZERO relabeling (P14 D-10): a success chip + humanized
`result_summary` for `:executed`, a failure chip + humanized reason + attempt count for
`:execution_failed` (color + text, brand token — never color-alone, brand §7.5). Wire the
ConversationLive reload via the existing thin-PubSub path (plain-assign, no streams — D16-12). Prove
that approver identity and policy snapshot are attributable after execute (OBS-02), and that the
per-proposal event timeline carries every attempt with actor attribution.

Purpose: Execution outcomes must be operator-legible and durable, and the durable trail must already
carry everything an optional Phase-17 evidence adapter would need — without building that adapter.

Output: Extended presenter clauses; ConversationLive execution-outcome handlers; headless presenter
tests + DB-backed OBS-02 attribution / rendered-chip proof.
</objective>
