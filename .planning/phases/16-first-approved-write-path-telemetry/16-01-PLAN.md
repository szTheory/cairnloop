<objective>
Build the first approved write path: the dedicated `ToolExecutionWorker` (the only place `run/3`
is ever called), the example `Cairnloop.Tools.InternalNote` governed-write tool, the contract
extensions that record execution outcomes (terminal statuses, execution event types, reserved
column population), the `Cairnloop.Governance` facade execute transition, and the additive resume
worker enqueue that hands off from `:execution_pending` to execution. Also lay the Wave 0 test
scaffold for the DB-backed proof.

Purpose: This is the milestone's first real side effect (ACT-01). Every prior phase deliberately
stopped short of `run/3`. The execution chain becomes a series of durable, independently-retryable
Oban jobs: `approve → resume (re-validate → :execution_pending) → execute (re-validate → run/3 →
record outcome)`. The phase is strictly ADDITIVE — no sealed path is churned.

Output: New worker + example tool + migrations; extended approval/event/proposal schemas; new
facade execute API; one additive line in the resume worker success branch; Wave 0 integration test
scaffold; extended fixtures.
</objective>
