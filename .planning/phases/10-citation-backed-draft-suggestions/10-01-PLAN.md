<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator entrypoints -> suggestion artifact | Untrusted entrypoint input becomes durable KB-maintenance state |
| retrieval evidence snapshot -> stored citation anchors | Suggestion evidence must preserve small, inspectable citation maps without hidden payloads |
| suggestion domain -> KB revision lineage | Revision suggestions must anchor to current published KB state, not optimistic UI assumptions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | T | `ArticleSuggestion` persistence | mitigate | Store suggestions in a new host-owned table instead of mutating `Draft` or published KB revisions, and persist full markdown rather than model-authored patches. |
| T-10-02 | I | embedded evidence snapshot | mitigate | Validate bounded `citation_target` and `metadata.destination` maps in `ArticleSuggestionEvidence` so only explicit citation anchors become durable operator-facing state. |
| T-10-03 | S | revision entrypoint anchor | mitigate | Resolve `base_revision_id` from `KnowledgeBase.get_latest_active_revision/1` and reject revision suggestions that cannot anchor to a published KB revision. |
</threat_model>
