<objective>
Build the approval transition facade (M011-S03-02): the public `Cairnloop.Governance` APIs that open
a lane, approve, reject, defer, and expire an approval тАФ each a durable decision co-committed with an
append-only `ToolActionEvent`, each guarded on the current `:pending` status, and `approve`
enqueuing the (now-existing) `ApprovalResumeWorker` without ever executing inline (APRV-01). Reject
and Defer require a persisted operator-visible reason (FLOW-03). Lane opening sets the
host-configurable TTL `expires_at` and schedules the `ApprovalExpiryWorker`.

Purpose: This is the operator's approve/reject/defer muscle memory made durable тАФ the "ReviewTask
for actions" transition surface. It depends on plan 15-01 (ToolApproval + get_active_approval) and
plan 15-02 (the workers it enqueues). Output: the narrow facade's approval transition APIs, with
`enqueue_fn` injection for headless tests; Wave 0 facade tests (approve-enqueues-never-executes,
append-only trail, transition-guarded-on-:pending, FLOW-03 reason gate) turn green.
</objective>

This plan creates on the `Cairnloop.Governance` facade:
- `request_approval(proposal, opts \\ [])` тАФ opens one `:pending` `ToolApproval` lane for a
  `:requires_approval` proposal; sets `expires_at = now + ttl` (host-configurable, finite default
  тЙИ 48h / `172_800`s тАФ document in @moduledoc); emits `:approval_requested` event; schedules
  `ApprovalExpiryWorker` (`scheduled_at: expires_at`, try/rescue). Returns `{:ok, %ToolApproval{}}`
  or surfaces the one-active-lane `unique_constraint` error.
- `approve(approval_id, actor_id, opts \\ [])` тАФ persists `:approved` decision + `:approved` event,
  THEN enqueues `ApprovalResumeWorker.new(%{"approval_id" => id})` via `Keyword.get(opts,
  :enqueue_fn, &safe_enqueue/1)`; never run/3. Guarded on current status `== :pending`.
- `reject(approval_id, actor_id, opts \\ [])` / `defer(approval_id, actor_id, opts \\ [])` тАФ
  REQUIRE `Keyword.fetch!(opts, :reason)` (or surface the decision_changeset reason error, FLOW-03);
  persist `:rejected`/`:deferred` decision + event; no enqueue. Guarded on `:pending`.
- `expire(approval_id, opts \\ [])` тАФ persists `:expired` decision + event (facade entry mirroring
  the worker flip, for completeness/admin). Guarded on `:pending`.
