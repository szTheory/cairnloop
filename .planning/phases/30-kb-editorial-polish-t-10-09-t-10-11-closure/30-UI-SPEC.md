> Visual and interaction contract for KB Editorial Polish + T-10-09 / T-10-11 Closure.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Phoenix LiveView with Tailwind + daisyUI + Heroicons |
| Preset | not applicable |
| Component library | daisyUI (vendored via `@plugin "../vendor/daisyui"`) |
| Icon library | Heroicons (via `@plugin "../vendor/heroicons"` — `hero-#{ICON}` classes) |
| Font | Atkinson Hyperlegible Next / Atkinson Hyperlegible (sans), Martian Mono (code) |

**Design system note:** This is an Elixir/Phoenix LiveView app. No shadcn/React component registry
applies. The shadcn gate is not applicable. The canonical brand tokens live in
`examples/cairnloop_example/assets/css/app.css` (landed in Phase 29) as `--cl-*` CSS custom
properties. All new render code uses `var(--cl-<token>)` exclusively — no inline hex fallbacks
(enforced by the negative-grep gate from BRAND-04).

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label padding, badge inner spacing |
| sm | 8px | Compact element spacing, filter list item gaps |
| md | 16px | Default section padding, aside internal padding |
| lg | 24px | Layout column gaps, section vertical margin |
| xl | 32px | Editor layout column gap (`gap-8`) |
| 2xl | 48px | Major surface section breaks |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Touch/click targets (buttons, action links): minimum 44px tall — apply `min-h-[44px]` to
  button elements per WCAG 2.5.5 guideline. Source: brand book §16.2.
- Nav shell height: 48px (`h-12`) — compact enough for a sub-header but clear enough to be
  a persistent context anchor across the four KB routes.

## Color

Source: brand book §7.2–7.4, §7.5. Tokens from Phase 29 `app.css` `:root` block.

| Role | Token | Hex (light) | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `var(--cl-bg)` | `#F5F0E6` | Page background (Trailpaper) |
| Secondary (30%) | `var(--cl-surface)` + `var(--cl-surface-raised)` | `#FBF7EE` / `#FFFFFF` | Cards, aside panels, editor panes, sidebar |
| Accent (10%) | `var(--cl-primary)` | `#A94F30` | Active nav item left marker, primary CTA buttons, focus ring |
| Destructive | `var(--cl-danger)` | `#B54C36` | Destructive confirmations, blocked/error states only |

Accent reserved for:
1. The shared editorial nav shell active-route left marker (2px `var(--cl-primary)` left border on current route).
2. "Create new article" primary CTA button (Path Copper fill, white text).
3. "Open for manual edit" and "Generate article suggestion" primary action buttons.
4. Focus rings on all interactive elements (`outline: 2px solid var(--cl-focus)`).
5. "Save Draft" / "Publish" primary action buttons in the Editor.

Secondary semantic tokens in use:
- `var(--cl-ai)` (`#7A5D78` Heather) — AI-origin evidence cards, grounding status indicator.
- `var(--cl-success)` (`#4A6238` Deep Lichen) — published/approved status chips.
- `var(--cl-info)` (`#3F6F80` Waypoint Blue) — evidence source citations, retrieval info.
- `var(--cl-warning)` (`#8B531E` Ember) — stale pressure labels, deferred/review-needed states.
- `var(--cl-text-muted)` (`#677066` Slate Lichen) — secondary metadata, timestamps, counts.

Accessibility rules (source: brand book §7.5):
- Never communicate state by color alone. Pair every color chip with a text label and/or icon.
- Normal text on `--cl-bg`: Basalt `#18211F` meets AAA 4.5:1+ contrast.
- Primary button: white `#FFFFFF` on Path Copper `#A94F30` meets AA 4.5:1.
- Muted text `#677066` on Trailpaper `#F5F0E6`: passes AA large text (3:1+); use only for
  supplementary labels at 13px+ weight 600, never for 15px body-weight text.

## Component Inventory

This section specifies new and modified components Phase 30 introduces. All are Phoenix LiveView
HEEx — no React, no shadcn.

### 1. Shared Editorial Nav Shell (KB-01)

**What:** A persistent horizontal sub-navigation rendered inside all four KB LiveViews
(`KnowledgeBase.Index`, `KnowledgeBase.Editor`, `SuggestionReview`, `KnowledgeBase.Gaps`).
Eliminates the current context-switch between unrelated layouts.

**Rendering approach:** A Phoenix function component (`def kb_nav/1`) defined in a shared module
(e.g., `Cairnloop.Web.KnowledgeBaseLive.NavComponent` or extracted as a plain `defp` in a shared
helper module). Each KB LiveView calls `<.kb_nav current={:index} />` (or `:editor`, `:suggestions`,
`:gaps`). This is purely additive — no sealed render code touched.

**Visual spec:**
- Container: `background: var(--cl-surface)`, `border-bottom: 1px solid var(--cl-border)`,
  `padding: 0 24px`, `height: 48px`, `display: flex`, `align-items: center`, `gap: 8px`.
- Nav items: links with `padding: 12px 16px`, `font-size: 13px`, `font-weight: 600`,
  `color: var(--cl-text-muted)`, `text-decoration: none`, `border-bottom: 2px solid transparent`,
  transition 120ms opacity.
- Active item: `color: var(--cl-text)`, `border-bottom: 2px solid var(--cl-primary)`.
- Hover (non-active): `color: var(--cl-text)`, `opacity: 0.85`.

**Labels (exact copy):**
- Knowledge base
- Suggestions
- Gaps

**Routes (exact):**
- `/knowledge-base`
- `/knowledge-base/suggestions`
- `/knowledge-base/gaps`

**Accessibility:** Use `<nav aria-label="Knowledge base">` as the wrapper. Mark the active link with
`aria-current="page"`. Keyboard focus ring: `outline: 2px solid var(--cl-focus); outline-offset: 2px`.

### 2. "Create new article" Button (KB-02)

**What:** A primary CTA button on `KnowledgeBase.Index` that routes to the Editor for a new article.
Currently absent.

**Visual spec:**
- Placement: top-right of the page header row, aligned with the "Knowledge Base" page title.
- Style: `background: var(--cl-primary)`, `color: var(--cl-primary-text)`, `border: none`,
  `border-radius: var(--cl-radius-sm)`, `padding: 8px 16px`, `font-size: 13px`, `font-weight: 600`,
  `min-height: 44px`, `cursor: pointer`.
- Hover: `opacity: 0.9`.
- Focus: `outline: 2px solid var(--cl-focus); outline-offset: 2px`.

**Label (exact copy):** "New article"

**Route:** Navigates to a new-article Editor path. Since `KnowledgeBase.Editor` currently only
accepts an existing article `id`, Phase 30 must handle the "new article" creation path. The button
triggers a LiveView event `"new_article"` which creates a blank article via the KnowledgeBase facade
and then `push_navigate` to `/knowledge-base/:id/edit`. Do not navigate to a bare `/new` path —
let the server create the record first so the editor always opens against a real DB record.

### 3. "View source gap" Sidebar (KB-03)

**What:** In `KnowledgeBase.Editor`, when opened via a `GapCandidate` handoff, a sidebar panel
surfaces the originating gap evidence in-context alongside the editor/preview split.

**Condition:** Rendered only when `@review_context.gap_candidate != nil` (a new assign introduced
in Phase 30 when a `gap_candidate_id` param is present alongside the handoff token).

**Visual spec (panel):**
- Position: third column to the right of the editor/preview split. On narrow viewports (below
  768px), collapse to a disclosure button `<details>` above the editor.
- Container: `width: 280px`, `flex-shrink: 0`, `background: var(--cl-surface)`,
  `border: 1px solid var(--cl-border)`, `border-radius: var(--cl-radius-md)`,
  `padding: 16px`, `overflow-y: auto`, `max-height: calc(100vh - 120px)`.
- Header: 13px / weight 600 / `color: var(--cl-text)` — label: "Source gap".
- Gap title: 15px / weight 600 / `color: var(--cl-text)`, margin-bottom 8px.
- Evidence count chip: inline chip — `background: var(--cl-surface-raised)`,
  `border: 1px solid var(--cl-border)`, `border-radius: var(--cl-radius-sm)`,
  `padding: 2px 8px`, `font-size: 12px`, `color: var(--cl-text-muted)`.
- Evidence items: each in a sub-card — `background: var(--cl-bg)`,
  `border: 1px solid var(--cl-border)`, `border-radius: var(--cl-radius-sm)`,
  `padding: 8px 12px`, `margin-bottom: 8px`.
  - Reason label: 12px / weight 600 / `color: var(--cl-warning)`.
  - Query excerpt: 13px / weight 400 / `color: var(--cl-text-muted)`.
  - Hit counts: 12px / `color: var(--cl-info)`.

**Accessibility:** The sidebar panel has `aria-label="Source gap evidence"`. It is not modal — the
operator can ignore it and edit freely.

### 4. "Open for manual edit" Affordance — Calm Copy (KB-04)

**What:** The `SuggestionReview` action button that currently delegates copy to
`ReviewTaskPresenter.action_label(:open_for_edit, task)`. This renders "Open for edit" or
"Open manual draft" depending on suggestion state. Phase 30 ensures the copy is calm,
reason-forward, and never leaks raw Elixir terms or raw JSON.

**Button copy contract:**

| Condition | Button label | Intent tooltip (aria-label) |
|-----------|-------------|---------------------------|
| Default `:open_for_edit` | "Open for manual edit" | "Open this suggestion in the article editor for manual revision" |
| `suggestion_type == :new_article` or `quick_fix_outcome == :blocked_manual_required` | "Create manual draft" | "Manually draft a new article based on this gap evidence" |
| Suggestion is `:failed` | "Review and draft manually" | "Inspect the generation failure and start a manual draft" |

**Copy constraints (brand book §5.1, §5.5):**
- Never render raw atom values (e.g., `:needs_manual_edit`, `:ready`) in operator-visible copy.
- Never render raw Elixir struct repr, Ecto changeset errors, or `inspect/1` output.
- Use the `ReviewTaskPresenter` and `ArticleSuggestionPresenter` layers exclusively for operator-facing text.
- All button labels: plain English, verb-first, ≤5 words.

**Visual spec:** Same button style as other review action buttons. No special destructive styling.
`background: var(--cl-surface-raised)`, `border: 1px solid var(--cl-border)`,
`border-radius: var(--cl-radius-sm)`, `padding: 8px 16px`, `font-size: 13px`, `font-weight: 600`,
`color: var(--cl-text)`, `min-height: 44px`.

### 5. Handoff Marker Interaction (SEC-01 + SEC-02)

**What:** No new visible UI element. The handoff marker (`manual_edit_opened_at` timestamp on the
`EditorHandoff` record) is a server-side security gate, not a visible component. However,
failure to pass the gate MUST produce a calm, brand-correct error surface, not a raw exception.

**Failure state UI contract:**

When `EditorHandoff.verify!/2` raises (i.e., the editor is reached without a valid handoff marker),
the LiveView must surface a flash error — not a 500 page, not a raw Elixir exception.

**Error flash copy (exact):**
> "This editor can only be opened from the review queue. Return to Suggestions and use 'Open for manual edit' to begin editing."

This copy satisfies:
- Brand book §5.2 "Errors: accountable, plain" tone.
- Brand book §5.6 error shape: (1) what happened — opened without handoff; (2) why it matters —
  security gate; (3) what to do next — return via review queue.
- Never leaks `:suggestion_id`, raw params, or Ecto error details.

**Redirect target on gate failure:** `/knowledge-base/suggestions` (the SuggestionReview index).

## Interaction Contracts

### KB-01 Nav Shell

- The nav shell renders above all KB LiveView content, below the host app's main navigation.
- Current route is visually indicated by `border-bottom: 2px solid var(--cl-primary)` AND
  `aria-current="page"` (never color alone — satisfies brand §7.5).
- Nav is purely presentational — does NOT hold state or trigger events. It uses `<.link navigate>`
  for full LiveView navigation.
- No loading spinner on nav link clicks — LiveView default page transition is sufficient.

### KB-02 New Article Button

- Single event `"new_article"` on `KnowledgeBase.Index`.
- On click: button shows disabled + `phx-click-loading` state while the server creates the
  article record (prevent double-submit). Copy stays "New article" during loading — no spinner
  copy change.
- On success: `push_navigate` to `/knowledge-base/:id/edit`.
- On error: `put_flash(:error, "Unable to create the article right now. Try again.")`.

### KB-03 Gap Sidebar Visibility

- Sidebar renders only when `@gap_candidate` is non-nil in Editor assigns.
- The handoff from `Gaps` LiveView must include `gap_candidate_id` in the signed handoff token.
  `EditorHandoff.sign/4` gains a `gap_candidate_id` optional field.
- Sidebar is read-only. No actions, no forms. Evidence is display-only.
- On mobile (`< 768px`): the sidebar collapses into a `<details>` summary with label
  "Source gap — tap to expand". Default: collapsed.

### KB-04 + SEC-01 + SEC-02 Handoff Gate

- The `open_for_manual_edit` event in `SuggestionReview` calls
  `EditorHandoff.sign/5` (extended to record `manual_edit_opened_at: DateTime.utc_now()`).
- `EditorHandoff.verify!/2` in `Editor.mount/3` checks that the decoded token contains a
  non-nil `manual_edit_opened_at`. If absent → raise → caught by a `mount` rescue clause →
  `put_flash(:error, <handoff gate message>)` + redirect to `/knowledge-base/suggestions`.
- The Editor MUST NOT preload `proposed_markdown` when `suggestion_id` is present in URL params
  but the handoff token is absent or invalid. The existing `load_suggestion/3` already calls
  `EditorHandoff.verify!/2`; Phase 30 adds the `manual_edit_opened_at` check inside that
  function's token payload.
- No visible security indicator is shown to the operator (this is a server-side gate).
  The only operator-visible consequence is the error flash + redirect on gate failure.

### Motion and Transitions

Source: brand book §15.

- Nav active-item transition: `transition: border-color 120ms ease, color 120ms ease`.
- Sidebar slide-in (when gap_candidate becomes non-nil): `opacity 0 → 1` over 160ms.
  No lateral slide — prefer fade (avoids layout shift).
- Button disabled state during `phx-click-loading`: `opacity: 0.6`, `cursor: not-allowed`.
- No bouncy, elastic, or sparkle animations anywhere on KB surfaces.
- Respect `prefers-reduced-motion`: wrap transitions in `@media (prefers-reduced-motion: no-preference)`.
